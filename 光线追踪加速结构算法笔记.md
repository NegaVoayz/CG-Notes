# 光线追踪加速结构（Ray Tracing Acceleration Structures）算法笔记

---

## 一、为什么需要加速结构？

### 问题
- 朴素光线追踪：每条光线与场景中**所有图元**求交
- 复杂度：`O(rays × primitives)`，百万级光线 × 百万级三角形 → 不可行

### 目标
- **快速剔除**不可能相交的物体
- **排序/组织**空间，使求交查询高效
- 利用**空间相干性**和**时间相干性**

---

## 二、加速结构分类

| 类别                         | 代表方法                                                   |
| ---------------------------- | ---------------------------------------------------------- |
| **基于物体（Object-based）** | 包围盒层次结构（BVH）                                      |
| **基于空间（Space-based）**  | 均匀网格（Uniform Grid）、八叉树（Octree）、KD-Tree、BSP树 |

---

## 三、包围体（Bounding Volume, BV）

### 核心思想
- 用简单几何体（如AABB、球体）**包围**复杂物体
- 先测试光线是否击中包围体
- 若未击中 → 直接跳过内部所有图元

### 优点
- 实现简单
- 剔除效率高

### 示例
```
if (ray hits bounding sphere)
    then test ray against internal triangles
else
    skip entirely
```

---

## 四、包围体层次结构（BVH, Bounding Volume Hierarchy）

### 数据结构
- **树形结构**
- 内部节点：存储包围体 + 指向子节点的指针
- 叶子节点：存储实际图元列表

### 构建算法（自底向上或自顶向下）
1. 计算当前节点所有图元的**包围盒**
2. 若图元数量 > 阈值，将图元分成两组
3. 递归构建左右子树
4. 更新当前节点的包围盒（包含所有子节点）

### 遍历算法（求交）
```
Intersect(node, ray):
    if ray misses node.bbox → return
    if node is leaf:
        for each primitive in node:
            compute intersection, update min_t
    else:
        Intersect(left_child)
        Intersect(right_child)
```

### 优化技巧
- **按距离排序子节点**，先处理近的（early termination）
- 若当前最近交点 `t` 小于子节点包围盒入口 `t`，可提前停止

### 复杂度
- 构建：`O(N log N)` 或 `O(N log² N)`
- 查询：平均 `O(log N)`，最坏 `O(N)`

## 五、均匀网格（Uniform Grid）

### 核心思想
- 将场景空间划分为**相同大小**的体素（voxel）
- 每个网格单元存储**与该单元相交的图元列表**

### 光线遍历
- 使用 **DDA（数字微分分析）** 算法沿光线方向逐格前进
- 只检测经过的网格单元中的图元

### 优点
- 实现简单
- 遍历速度快（增量计算）

### 缺点
- 网格分辨率难以选择
  - 太粗 → 每个格子太多图元
  - 太细 → 遍历太多空格子

### 分辨率启发式
```
#cells = C × #objects
C ≈ 27（3D场景经验值）
```

### 适用场景
- 物体**分布均匀**且**大小相近**（如粒子系统）

## 六、八叉树（Octree）

### 核心思想
- 递归地将**立方体**空间均匀划分为 **8 个子立方体**（octants）
- 每个节点存储与其重叠的图元
- 若子节点为空或无图元 → 停止划分

### 光线遍历
- 从根节点开始
- 递归进入与光线相交的子节点
- 需处理**邻居节点查找**（比网格复杂）

### 优点
- 自适应 → 稀疏区域少划分
- 内存比均匀网格小

### 缺点
- 遍历逻辑比网格复杂
- 划分必须是**轴对齐立方体**，灵活性受限

---

## 七、KD-Tree（K-Dimensional Tree）

### 核心思想
- **空间二分**：每次用**一个轴对齐平面**将空间分为两部分
- 交替或自适应选择划分轴（x, y, z）
- 内部节点只存划分信息，**不存图元**
- 叶子节点存图元列表

### 构建算法
```
Build(nodes):
    if node contains few primitives → mark as leaf
    else:
        choose split axis and position
        partition primitives into left/right
        recursively build children
```

### 划分策略
- 选择**空间居中**或**物体居中**
- 常用 **SAH（表面积启发式）** 优化划分位置

### 光线遍历
- 从根开始
- 判断光线与左右子节点的包围盒相交顺序
- 先处理近的子节点（early termination）

### 优点
- 高效，广泛用于离线渲染
- 自适应空间划分

### 缺点
- 构建成本较高
- 动态场景更新困难

---

## 八、BSP树（Binary Space Partitioning Tree）

### 核心思想
- 与KD-Tree类似，但**划分平面不限于轴对齐**
- 可以是任意方向的平面

### 构建
- 递归选择平面，将空间分成两个凸多面体
- 每个节点存储平面方程 + 子节点

### 光线遍历
- 递归进入与光线相交的子空间
- 支持**前后排序**，便于透明物体渲染

### 优点
- 更灵活，可适配任意几何

### 缺点
- 构建复杂度高
- 遍历计算开销大（非轴对齐求交）

---

## 九、加速结构性能对比

| 结构     | 构建时间 | 查询效率 | 内存 | 动态场景 | 实现复杂度 |
| -------- | -------- | -------- | ---- | -------- | ---------- |
| 均匀网格 | 低       | 中       | 高   | 中       | 低         |
| 八叉树   | 中       | 中高     | 中   | 中       | 中         |
| KD-Tree  | 高       | 高       | 中   | 差       | 高         |
| BVH      | 中       | 高       | 低   | 好       | 中         |
| BSP树    | 高       | 中高     | 中   | 差       | 高         |

## 十、其他加速方法（补充）

| 方法                         | 说明                                        |
| ---------------------------- | ------------------------------------------- |
| **Screen-space coherence**   | 相邻像素光线可能击中同一物体 → 缓存上次命中 |
| **Beam/Pencil/Cone tracing** | 用光线束代替单根光线，一次覆盖多个像素      |
| **Early termination**        | 找到足够近的交点后，跳过更远的物体          |
| **并行化**                   | 每条光线独立 → 高度并行（GPU友好）          |
