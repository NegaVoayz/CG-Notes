## 1. 传统图形管线（按顺序）

### 1.1 输入装配器 (IA)
- **输入**：Vertex Buffer（顶点数据） + Index Buffer（索引数据）
- **操作**：根据索引装配成**图元（点/线/三角形）**
- **输出**：顶点流（仍在**局部空间**，未做任何变换）

---

### 1.2 顶点着色器 (VS)
- **输入**：局部空间顶点
- **操作**：
  - 应用 `Model Matrix`（局部 → 世界）
  - 应用 `View Matrix`（世界 → 观察空间）
  - 可选：逐顶点光照、法线变换
- **输出**：观察空间顶点（`w = 1`，线性空间）
- **关键**：此时仍未做投影，保证后续细分无扭曲

---

### 1.3 曲面细分着色器 (TCS + TES) —— 可选
- **位置**：VS 之后，GS 之前
- **操作**：
  - **TCS**：计算细分因子（每条边/内部切几段）
  - **TES**：根据 UVW 坐标生成新顶点位置
- **输出**：仍为**观察空间**顶点（保持线性插值）
- **用途**：LOD、地形、位移贴图

---

### 1.4 几何着色器 (GS) —— 可选
- **输入**：一个图元（观察空间）
- **操作**：
  - 可**改变图元类型**（点 → 三角形）
  - 可**改变图元数量**（扩增或丢弃）
  - 可**计算面法线**
- **输出**：0~N 个图元（观察空间）
- **注意**：现代引擎中常用 Compute Shader 或 Mesh Shader 替代 GS

---

### 1.5 顶点后处理 (VPP) —— 固定功能
- **位置**：GS 之后，光栅化之前
- **子步骤**：
  1. **投影变换**：应用 `Projection Matrix`，产生 `w` 分量 → **进入裁剪空间**
  2. **裁剪**：剔除 `[-w, w]` 范围外的图元，在边界处生成新顶点
  3. **透视除法**：`(x, y, z) / w` → **进入 NDC**
  4. **视口变换**：NDC → **屏幕坐标（像素）**
- **关键**：这是**最后一个三维空间阶段**，之后数据进入二维像素空间

---

### 1.6 光栅化 —— 固定功能
- **输入**：屏幕空间三角形顶点
- **操作**：
  - 计算三角形覆盖的像素
  - 对每个像素进行**重心坐标插值**（法线、UV、颜色等）
- **输出**：像素片段（Fragment），含插值属性
- **注意**：插值使用**透视校正**（乘以 `1/w`）

---

### 1.7 逐片段早期测试 (Early Per-Fragment Tests)
- **位置**：Fragment Shader 之前
- **操作**：
  - **深度测试**：比较当前片段与深度缓冲
  - **模板测试**：比较模板值
- **效果**：被遮挡片段直接丢弃，**不执行 Fragment Shader**，大幅节省性能
- **限制**：若 FS 会修改深度或使用 `discard`，则该优化失效

---

### 1.8 片段着色器 (FS)
- **输入**：插值后的属性（UV、法线、颜色等）
- **操作**：
  - **纹理采样**（UV）
  - **光照计算**（Blinn-Phong、PBR）
  - **阴影判断**
- **输出**：像素颜色（RGBA），可选修改深度值
- **性能**：**管线中最耗时的阶段**，分辨率越高开销越大

---

### 1.9 逐片段后期测试 (Late Per-Fragment Tests)
- **位置**：Fragment Shader 之后
- **触发条件**：FS 中使用了 `discard` 或修改了 `gl_FragDepth`
- **操作**：执行深度/模板测试，丢弃可见性变更后失效的片段
- **代价**：被遮挡的片段已经执行完 FS，**浪费大量性能**

---

### 1.10 颜色混合 (Blending)
- **输入**：FS 输出的颜色 + 帧缓冲中的已有颜色
- **公式**：
  ```
  final = src × srcFactor + dst × dstFactor
  ```
- **常见模式**：
  - **半透明**：`SRC_ALPHA / ONE_MINUS_SRC_ALPHA`
  - **叠加发光**：`ONE / ONE`
- **关键**：半透明物体需**从远到近排序**绘制，保证正确混合
- **注意**：Blending 不处理深度，只处理颜色和透明度

## 2. 现代替代管线

### 2.1 Mesh Shader Pipeline（替代 IA + VS + TCS + TES + GS）
- **Task Shader（可选）**：
  - 决定是否启动 Mesh Shader（视锥剔除 / LOD）
  - 决定启动多少工作组
- **Mesh Shader**：
  - 类似 Compute Shader，**直接生成顶点和图元**
  - 可读 Storage Buffer 作为输入
  - 输出：观察空间或世界空间顶点
- **优点**：
  - 绕过 IA / VS / 细分 / GS 的固定限制
  - 更灵活的剔除与生成逻辑
  - 更高并行度

---

### 2.2 Compute Shader（完全可编程通用计算）
- **不参与图形管线**，独立执行
- **用途**：
  - 粒子系统更新
  - 后处理特效（模糊、HDR）
  - Ray Query（光线追踪）
  - 间接绘制参数生成（GPU-Driven Rendering）
- **特点**：完全由开发者控制线程模型和内存访问

---

## 3. 补充：坐标空间变迁路径

| 阶段          | 空间     | 维度                      | 描述                   |
| :------------ | :------- | :------------------------ | :--------------------- |
| IA → VS 输入  | 局部空间 | 3D                        | 模型原点坐标           |
| VS 输出       | 观察空间 | 3D（`w=1`）               | 相机为原点的线性空间   |
| TES / GS 输出 | 观察空间 | 3D（`w=1`）               | 保持线性，便于插值     |
| 投影变换后    | 裁剪空间 | 齐次 4D                   | 产生 `w`，用于透视除法 |
| 透视除法后    | NDC      | 3D（`[-1,1]` 或 `[0,1]`） | 标准化坐标             |
| 视口变换后    | 屏幕空间 | 2D（像素）                | 最终显示坐标           |

---

## 4. 关键优化建议（重点记忆）

1. **尽量启用 Early Z Test**：
   - 不在 FS 中写 `gl_FragDepth`
   - 避免不必要的 `discard`
2. **半透明物体排序渲染**：
   - 先画不透明物体（填满深度缓冲）
   - 从远到近画半透明物体
   - 半透明物体关闭深度写入
3. **尽量减少使用 GS**：
   - 推荐用 Compute Shader 或 Mesh Shader 替代
4. **尽量使用 GPU Instancing**：
   - 多个相同模型用 `gl_InstanceIndex` + Storage Buffer 实现
