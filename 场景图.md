# 场景图

**场景图**是计算机图形学中用于**组织和存储三维场景中所有物体及其空间关系**的数据结构。它本质上是一个**有向无环图**（通常是树），其核心思想是通过**层次化的坐标变换**来管理复杂的场景。

## 一、基本结构

场景图由**节点**（Node）和**边**（Edge）组成：

- **根节点（Root）**：整个场景的顶层，通常代表世界坐标系。
- **组节点（Group Node）**：包含子节点，用于组织层次结构，可以存储变换矩阵。
- **叶节点（Leaf Node）**：包含实际的几何体（网格）、光源、相机等可渲染对象。

每个节点都保存一个**相对于父节点的变换矩阵**（平移、旋转、缩放）。

```
        Root (世界坐标系)
         |
    +----+----+
    |         |
  Group A   Group B
 (位置: 5,0,0) (位置: 0,0,3)
    |         |
  Sphere    Group C
            (旋转: 45°)
              |
            Cube
```

## 二、工作原理

### 1. 组合变换（Composition）

当从根节点遍历到某个叶节点时，需要**累积**路径上所有节点的变换矩阵：

$$
M_{world} = M_{parent} \times M_{local}
$$
这确保了子物体的位置、朝向是基于父物体定义的，实现了“相对坐标”的概念。

### 2. 遍历（Traversal）

渲染流程通常采用**深度优先遍历**：

1. 从根节点开始。
2. 应用当前节点的变换矩阵。
3. 递归渲染所有子节点。
4. 恢复父节点的变换（通过栈保存状态）。

## 三、核心优势

### 1. 实例化（Instancing）

同一个几何体可以在不同位置多次出现，只需创建多个引用节点，各带不同变换矩阵。这**节省了内存**，也便于批量更新。

### 2. 层次化建模

构建复杂对象非常自然：

- 人体模型：躯干（父节点）→ 上臂（子节点，绕肩旋转）→ 前臂（孙节点，绕肘旋转）
- 机械臂、车辆、太阳系等都可按层次建模。

### 3. 局部坐标与独立性

每个物体在其局部坐标系中定义，与场景中其他物体无关。移动父节点时，所有子节点自动跟随。

### 4. 简化碰撞检测与拾取

通过层次包围盒（BVH）加速：若父节点未命中，则无需检查其子节点。

## 四、典型应用

| 应用领域                        | 说明                                              |
| :------------------------------ | :------------------------------------------------ |
| **游戏引擎**（Unity、Unreal）   | 场景层级窗口（Hierarchy Panel）即场景图的直观体现 |
| **3D建模软件**（Blender、Maya） | 大纲视图（Outliner）管理物体父子关系              |
| **机器人仿真**（ROS、Gazebo）   | 描述关节链与刚体运动学                            |
| **CAD系统**                     | 装配体结构，部件之间相对定位                      |
| **Web 3D**（Three.js）          | `THREE.Group` 和 `THREE.Object3D` 即场景图实现    |

## 五、重要注意事项

### 1. 变换顺序不可交换

矩阵乘法的顺序至关重要。通常默认按 **缩放 → 旋转 → 平移**（SRT）顺序组合。

### 2. 全局坐标 vs 局部坐标

`object.worldMatrix`（全局） = `parent.worldMatrix` × `object.localMatrix`。

获取物体在世界空间中的位置：

```
worldPosition = worldMatrix * localPosition
```

### 3. 性能考量

- **深度优先遍历**有递归开销，大型场景需优化（如遮挡剔除）。
- **动态更新**：若父节点移动，所有子节点的世界矩阵需要重新计算（“脏标记”机制）。

### 4. 图的通用性

场景图不一定严格是树。有时允许 **DAG（有向无环图）** 实现资源共享，但需避免循环引用导致无限递归。

## 七、与其他概念的关系

| 概念                       | 关系                                           |
| :------------------------- | :--------------------------------------------- |
| **变换矩阵**               | 场景图中每个节点存储的数学基础                 |
| **包围体层次（BVH）**      | 常构建在场景图之上，用于加速渲染和碰撞检测     |
| **空间划分（八叉树/BSP）** | 另一种场景管理方法，适用于室外大场景或静态场景 |

---

**总结**：场景图是一种优雅的**递归数据结构**，通过相对变换实现了复杂场景的高效管理和渲染。它是现代图形系统（从游戏引擎到CAD软件）的基石，理解了场景图，就等于掌握了3D场景组织的“骨架”。
