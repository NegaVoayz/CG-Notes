## 1. 渲染管线概述

**渲染**：从几何图元（点、线、三角形）生成图像的过程。

核心问题：
- 场景表示（几何体、材质、光源）
- 观察者（相机）表示
- 可见面判断
- 光照模拟
- 硬件管线实现

## 2. 传统图形管线各阶段

![Vulkan Pipeline Diagram](Vulkan%20Pipeline%20Diagram.png)

### 2.1 输入装配器 (Input Assembler, IA)
- **输入**：Vertex Buffer（顶点数据）+ Index Buffer（索引数据）
- **操作**：根据索引装配成图元（点/线/三角形）
- **输出**：顶点流（仍在**局部空间**）

### 2.2 顶点着色器 (Vertex Shader, VS)
- **输入**：局部空间顶点
- **操作**：
  - 应用 Model Matrix（局部 → 世界）
  - 应用 View Matrix（世界 → 观察空间）
  - 可选：逐顶点光照、法线变换
- **输出**：观察空间顶点（$w=1$，线性空间）
- **关键**：此时仍未做投影，保证后续细分无扭曲

### 2.3 曲面细分着色器 (Tessellation) — 可选
- **位置**：VS 之后、GS 之前
- **TCS** (Tessellation Control Shader)：计算细分因子（每条边/内部切几段）
- **TES** (Tessellation Evaluation Shader)：根据 UVW 坐标生成新顶点位置
- **输出**：仍为**观察空间**顶点
- **用途**：LOD、地形、位移贴图

### 2.4 几何着色器 (Geometry Shader, GS) — 可选
- **输入**：一个图元（观察空间）
- **操作**：可改变图元类型和数量（点→三角形、扩增、丢弃）
- **输出**：0~N 个图元（观察空间）
- **注意**：现代引擎常用 Compute Shader 或 Mesh Shader 替代（GS 性能不佳）

### 2.5 顶点后处理 (Vertex Post-Processing, VPP) — 固定功能
1. **投影变换**：应用 Projection Matrix，产生 $w$ 分量 → 进入裁剪空间
2. **裁剪**：剔除 $[-w,w]$ 范围外的图元，在边界处生成新顶点
3. **透视除法**：$(x,y,z)/w$ → 进入 NDC
4. **视口变换**：NDC → 屏幕坐标（像素）

> 这是**最后一个 3D 空间阶段**，之后进入 2D 像素空间。

### 2.6 光栅化 (Rasterization) — 固定功能
- **输入**：屏幕空间三角形顶点
- **操作**：
  - 确定三角形覆盖的像素
  - 对每个像素进行**重心坐标插值**（法线、UV、颜色等）
- **输出**：像素片段 (Fragment)，含插值属性
- **注意**：插值使用**透视校正**（乘以 $1/w$）

### 2.7 早期逐片段测试 (Early Per-Fragment Tests)
- **位置**：Fragment Shader **之前**
- **操作**：深度测试 + 模板测试
- **效果**：被遮挡片段直接丢弃，**不执行 Fragment Shader** → 大幅节省性能
- **限制**：若 FS 会修改深度或使用 `discard`，该优化失效

### 2.8 片段着色器 (Fragment Shader, FS)
- **输入**：插值后的属性（UV、法线、颜色等）
- **操作**：纹理采样、光照计算、阴影判断
- **输出**：像素颜色 (RGBA)，可选修改深度值
- **性能**：管线中最耗时的阶段，分辨率越高开销越大

### 2.9 后期逐片段测试 (Late Per-Fragment Tests)
- **触发条件**：FS 中使用了 `discard` 或修改了深度
- **代价**：被遮挡的片段已执行完 FS，浪费大量性能

### 2.10 颜色混合 (Blending)
- **输入**：FS 输出颜色 + 帧缓冲已有颜色
- **公式**：`final = src × srcFactor + dst × dstFactor`
- **常见模式**：
  - 半透明：`SRC_ALPHA / ONE_MINUS_SRC_ALPHA`
  - 叠加发光：`ONE / ONE`
- **关键**：半透明物体需**从远到近排序**绘制

## 3. 坐标空间变迁路径

> ⚠️ 完整坐标变迁是必考题，理解每个空间的维度和含义。

| 阶段 | 空间 | 维度 | 说明 |
| :--- | :--- | :--- | :--- |
| IA → VS 输入 | 局部空间 (Local) | 3D | 模型原点坐标 |
| VS 输出 | 观察空间 (View) | 3D ($w=1$) | 相机为原点的线性空间 |
| TES/GS 输出 | 观察空间 (View) | 3D ($w=1$) | 保持线性，便于插值 |
| 投影变换后 | 裁剪空间 (Clip) | 齐次 4D | 产生 $w$，用于透视除法 |
| 透视除法后 | NDC | 3D | $[-1,1]$ (GL) 或 $[0,1]$ (VK) |
| 视口变换后 | 屏幕空间 (Screen) | 2D (像素) | 最终显示坐标 |

### 透视校正插值

屏幕空间线性插值 ≠ 3D 空间线性插值。正确做法是先对 $u/w$ 和 $v/w$ 在屏幕空间线性插值，再除以 $1/w$ 的插值结果（现代 GPU 自动完成）：
$$u = \frac{(u/w)_{interp}}{(1/w)_{interp}}$$

## 4. 场景图 (Scene Graph)

### 4.1 定义与结构
**场景图**：用于组织和存储三维场景中所有物体及其空间关系的有向无环图（DAG）。

- **根节点 (Root)**：世界坐标系
- **组节点 (Group Node)**：包含子节点，存储变换矩阵
- **叶节点 (Leaf Node)**：包含实际几何体、光源、相机

```
        Root (世界坐标系)
         |
      +--+--+
      |     |
    Group  Group
   (平移)  (旋转)
      |     |
    Mesh   Group
            |
           Mesh
```

### 4.2 组合变换
世界变换 = 父节点世界变换 × 本节点局部变换：
$$M_{world} = M_{parent} \cdot M_{local}$$

### 4.3 核心优势

| 优势 | 说明 |
| :--- | :--- |
| **实例化** | 同一几何体在不同位置多次出现（不同变换矩阵） |
| **层次化建模** | 人体骨架、机械臂等父子关系 |
| **局部坐标独立性** | 物体在其局部坐标系定义，移动父节点子节点自动跟随 |
| **简化碰撞检测** | 父节点包围盒未命中 → 无需检查子节点 |

### 4.4 注意事项
- 变换顺序不可交换（通常按 SRT：缩放→旋转→平移）
- 深度优先遍历有递归开销
- 动态更新需"脏标记"机制避免重复计算

## 5. 现代替代管线

### 5.1 Mesh Shader Pipeline
**替代**：IA + VS + TCS + TES + GS 全部

- **Task Shader**（可选）：决定是否启动 Mesh Shader（视锥剔除/LOD），决定启动多少工作组
- **Mesh Shader**：类似 Compute Shader，直接生成顶点和图元

**优点**：绕过固定限制，更灵活的剔除与生成逻辑，更高并行度

### 5.2 Compute Shader
**不参与图形管线**，独立执行：
- 粒子系统更新
- 后处理特效（模糊、HDR）
- 间接绘制参数生成 (GPU-Driven Rendering)

## 6. Vulkan 内存与资源对象

### 6.1 Buffer（线性缓冲区）
> 本质：一段连续的、无格式的显存。存放"二进制数组"。

| 名称 | 存储内容 | 着色器访问 | 典型用途 |
| :--- | :--- | :--- | :--- |
| **Vertex Buffer** | 顶点属性数组 | IA 读取 | 几何体模板 |
| **Index Buffer** | 顶点索引数组 | IA 读取 | 图元装配顺序 |
| **Uniform Buffer** | 小量结构化数据（MVP、光源） | `uniform` 块 | 每帧常量数据 |
| **Storage Buffer** | 大量结构化数据 | `buffer` 块 | 可读写海量数据 |
| **Indirect Buffer** | 绘制命令参数 | GPU 读取 | GPU-Driven 绘制 |
| **Push Constant** | 小量实时参数 | 指令流内嵌 | 极速更新 |

### 6.2 Image（图像缓冲区）
> 本质：带格式的、支持多维寻址的显存。存放"像素/纹素"。

| 名称 | 存储内容 | 特性 | 典型用途 |
| :--- | :--- | :--- | :--- |
| **Sampled Image** | 只读纹理 | 支持滤波、Mipmap | 漫反射贴图、法线贴图 |
| **Storage Image** | 可读写纹理 | 无滤波，精确访问 | 后处理、Compute Shader 中间结果 |
| **Color Attachment** | 渲染目标 | 支持混合 (Blending) | 帧缓冲、G-Buffer |
| **Depth/Stencil Attachment** | 深度+模板值 | 硬件控制读写 | Z-Buffer、Shadow Map |
| **Input Attachment** | 前一个 Subpass 输出 | 同 Render Pass 内零拷贝 | 延迟渲染 G-Buffer 读取 |

### 6.3 CUDA 类比速查

| Vulkan | CUDA |
| :--- | :--- |
| Uniform Buffer | `cudaMalloc` + 只读指针（类似 `__constant__`） |
| Storage Buffer | `cudaMalloc` + 裸指针 |
| Sampled Image | `cudaTextureObject_t` |
| Storage Image | `cudaSurfaceObject_t` 或指针 |
| Push Constants | Kernel 直接参数 |
| Descriptor Sets | GPU 内存地址表 |

## 7. Descriptor Set

> **本质**：GPU 的"虚拟地址表"，告诉硬件"这块显存是什么、怎么用"。本身不存数据，只存指针和访问规则。

| 描述符类型 | 绑定的资源 |
| :--- | :--- |
| `UNIFORM_BUFFER` | Uniform Buffer |
| `STORAGE_BUFFER` | Storage Buffer |
| `SAMPLED_IMAGE` | Sampled Image |
| `STORAGE_IMAGE` | Storage Image |
| `INPUT_ATTACHMENT` | Input Attachment |
| `SAMPLER` | Sampler（采样器） |

## 8. 固定管线 vs 可编程管线

| 维度 | 固定管线 (1990s) | 可编程管线 (2001+) |
| :--- | :--- | :--- |
| **变换/光照** | 硬件写死算法 | 顶点着色器可编程 |
| **像素着色** | 固定纹理混合 | 片段着色器图灵完备 |
| **几何处理** | 无 | GS/TCS/TES/Mesh Shader |
| **通用计算** | 不支持 | Compute Shader |
| **代表 API** | OpenGL 1.x, D3D9 Fixed | OpenGL 3.3+, Vulkan, D3D12 |
| **灵活度** | 极低 | 任意算法 |

### Draw Call 的 CPU-GPU 交互

1. **CPU**：应用设置渲染状态（Shader、Buffer 绑定、Pipeline State）
2. **CPU**：发出 Draw Call（`vkCmdDraw` / `glDrawElements`）
3. **GPU**：Command Processor 解析 → 分发到各管线阶段
4. Draw Call 开销主要来自 CPU 端的验证和状态切换，**减少 Draw Call** = GPU Instancing / Indirect Draw

## ⚠️ 常见考点

1. **完整管线阶段的顺序和功能** → 填图/排序题
2. **Early Z Test 何时失效？** → FS 中写深度或用 `discard` 时
3. **透视校正插值的原理** → 对 $u/w$ 和 $1/w$ 插值，再除回去
4. **场景图中子节点世界矩阵如何计算？** → $M_{world} = M_{parent} \cdot M_{local}$
5. **Uniform Buffer vs Storage Buffer 的区别？** → UB 小而快（只读常量），SB 大（可读写）
6. **Descriptor Set 的作用？** → GPU 内存地址表，告诉硬件资源在哪、怎么用
7. **Mesh Shader 替代了哪些传统阶段？** → IA + VS + TCS + TES + GS
8. **半透明物体为什么需要排序？** → Blending 依赖帧缓冲已有颜色，需从远到近保证混合正确
