# 纹理映射与滤波

纹理映射（Texture Mapping）是让 3D 模型看起来真实的关键技术。简单来说，就是把一张 2D 图片（纹理）“贴”到 3D 模型的表面上。

这个过程主要分为两步：**映射（找到位置）** 和 **滤波（算出颜色）**。

---

### 第 1 步：纹理映射（找到位置）

这一步要解决的核心问题是：**“模型表面的这个 3D 点，对应纹理图片上的哪个 2D 点？”**

#### 映射方式（最主流：UV 映射）
在建模软件中，美术会把模型的表面“剪开”并“摊平”到一张 2D 图片上。这样，模型的每个顶点都会获得一个 `(u, v)` 坐标。

-   **u**：水平方向（0 ~ 1）。
-   **v**：垂直方向（0 ~ 1）。

**渲染时**：顶点着色器（Vertex Shader）把顶点的 UV 坐标传递给像素着色器（Fragment Shader）。光栅化阶段会自动插值，计算出三角形内部每个像素的 UV 坐标。像素着色器拿到这个坐标后，去纹理图片中**采样**颜色。

> **除了 UV 映射外，还有其他方式**：
> -   **环境映射**：用于反射/折射，通过法线方向去 CubeMap（立方体纹理）里采样。
> -   **投影映射**：像投影仪一样把纹理投射到物体上，常用于阴影贴图（Shadow Map）。

---

### 第 2 步：纹理滤波（算出颜色）

这是核心问题。纹理是**离散的像素（Texel）**，而 UV 坐标是**连续的浮点数**。当 UV 坐标恰好落在两个像素之间时，我们该如何取值？

根据场景不同，我们有三种策略：

#### 1. 放大滤波（Magnification）—— 纹理太小，物体太大
当纹理像素（Texel）小于屏幕像素时，一个屏幕像素需要覆盖纹理中的一个小区域。

-   **Nearest（最近邻）**：直接取离 UV 坐标最近的纹理像素。**效果**：马赛克。
-   **Bilinear（双线性插值）**：取周围 **2x2** 个纹理像素，进行加权平均。**效果**：边缘模糊但平滑。

> 这是**滤波**的核心——用周围像素的加权平均来算出非整数位置的平滑颜色。

#### 2. 缩小滤波（Minification）—— 纹理太大，物体太小
当纹理像素（Texel）小于屏幕像素时，一个屏幕像素需要覆盖纹理中的一大片区域。如果只是简单采样，会产生严重的**闪烁和锯齿**。

-   **Mipmap（多级渐远纹理）**：预先生成一系列逐渐缩小的纹理图（每级分辨率减半），然后根据物体距离自动选择或混合使用合适的层级。
-   **Trilinear（三线性插值）**：在两层相邻的 Mipmap 层级之间再做一次插值，消除层级切换时的“断层”。

#### 3. 各向异性滤波（Anisotropic Filtering）—— 处理斜视问题
当观察角度很斜（比如从侧面看地面），纹理在水平和垂直方向上的缩放比例差异巨大，普通的 Mipmap 会让远处或斜角部分变得模糊。

-   **各向异性滤波**：沿观察方向**非均匀采样**，解决斜视角度的模糊问题。这几乎是目前游戏画质的标配。

---

### ⚠️ 一个常见误区：UV 坐标与纹理尺寸无关

纹理滤波发生在**像素着色器（Fragment Shader）** 中，它接受的 UV 坐标始终是 **0~1** 的浮点数。硬件会根据当前纹理的**实际分辨率**（如 1024x1024 或 512x512）自动换算成像素坐标，并执行相应的滤波算法。

---

### 💡 总结

| 阶段         | 发生位置              | 核心任务                                                     |
| :----------- | :-------------------- | :----------------------------------------------------------- |
| **UV 映射**  | 建模软件 + 顶点着色器 | 建立 3D 点与 2D 图片的对应关系                               |
| **纹理采样** | 像素着色器            | 根据 UV 坐标读取纹素（Texel）                                |
| **滤波**     | 像素着色器 / 硬件单元 | 通过插值（双线性/三线性）或各向异性采样，处理坐标不对齐时的颜色平滑问题 |
