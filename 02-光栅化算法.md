## 1. 扫描转换 (Scan Conversion) 概述

**扫描转换（光栅化）**：将连续图形 → 离散像素的过程。

核心挑战：
- 如何选择"最接近"理想图形的像素
- 如何处理边界情况（端点、顶点、共享边）
- 如何高效增量计算（避免浮点、避免重复）

## 2. 线段绘制

### 2.1 问题建模

给定两端点 $(x_0, y_0)$ 和 $(x_1, y_1)$（整数坐标），确定中间哪些像素最接近理想直线。

**关键约束**（第一象限 $0 < m < 1$ 情形）：
- x 方向为主步进方向（每次 x+=1）
- y 方向每次要么不变，要么 +1
- 问题简化为：**每走一列，y 要不要 +1？**

### 2.2 DDA 算法 (Digital Differential Analyzer)

基本思路：增量更新 y 值
$$y_{k+1} = y_k + m$$

**缺点**：涉及浮点数运算和取整，可能累积误差。

### 2.3 Bresenham 算法

#### 核心优势
**仅使用整数加法、减法和移位**，无浮点运算，硬件实现效率极高。

#### 数学推导

设当前像素 $(x_k, y_k)$，下一个 x 必然为 $x_k + 1$，真实 y 值为 $y = m(x_k + 1) + b$。

两个候选像素到理想位置的距离差：
$$d_1 - d_2 = 2m(x_k + 1) - 2y_k + 2b - 1$$

乘以 $\Delta x$ 消去浮点数，定义**决策参数** $p_k$：
$$p_k = \Delta x \cdot (d_1 - d_2) = 2\Delta y \cdot x_k - 2\Delta x \cdot y_k + C$$

决策规则：
- $p_k < 0$：选下方像素（E，东），$y$ 不变
- $p_k \geq 0$：选上方像素（NE，东北），$y$ +1

#### 增量递推（核心优化）

$$p_{k+1} = \begin{cases} p_k + 2\Delta y & \text{if } p_k < 0 \text{ (选E)} \\ p_k + 2\Delta y - 2\Delta x & \text{if } p_k \geq 0 \text{ (选NE)} \end{cases}$$

初始值：$p_0 = 2\Delta y - \Delta x$

#### 伪代码（第一象限，0 < m < 1）

```
plotLine(x0, y0, x1, y1):
    dx = x1 - x0;  dy = y1 - y0
    p = 2*dy - dx          // 初始决策值
    y = y0
    for x = x0 to x1:
        plot(x, y)
        if p < 0:
            p = p + 2*dy   // 选E，y不变
        else:
            y = y + 1
            p = p + 2*dy - 2*dx   // 选NE
```

#### 推广到任意斜率（通用实现）

```cpp
void drawLine(int x1, int y1, int x2, int y2) {
    int dx = abs(x2 - x1), dy = abs(y2 - y1);
    int sx = (x1 < x2) ? 1 : -1;  // x方向步进
    int sy = (y1 < y2) ? 1 : -1;  // y方向步进
    int err = dx - dy;            // 初始误差

    while (true) {
        plot(x1, y1);
        if (x1 == x2 && y1 == y2) break;
        int e2 = 2 * err;
        if (e2 > -dy) { err -= dy; x1 += sx; }
        if (e2 <  dx) { err += dx; y1 += sy; }
    }
}
```

#### 优化路径总结

| 步骤 | 方法 | 问题 |
| :--- | :--- | :--- |
| 1 | 每次 $$y = m x + b$$ | 浮点乘除，慢 |
| 2 | 每次 $$y = y + m$$ | 仍需浮点累加 |
| 3 | y 拆为整数+小数，$$f = f + m$$ | 小数比较 $$f > 0.5$$ |
| 4 | f 乘以 dx | 比较变为 $$f > dx/2$$ |
| 5 | f 再乘以 2 | 比较变为 $$f > dx$$，无除法 |
| 6 | 初始 f = -dx（平移） | 比较变为 $$f > 0$$，最快 |

> **最终版本**：循环内只需整数比较和加法，零乘除。

### 2.4 Bresenham 画圆算法

> **考点：** 常见考题：推导或手算 Bresenham 画圆算法的决策参数。

画圆的关键：利用**八对称性 (Eight-fold Symmetry)**，只需计算第一象限中斜率 $-1 < m < 0$ 的一个八分圆（45° 到 90°），其余通过对称得到。

决策参数 $p_k$ 判断下一个像素选 $(x_k+1, y_k)$ 还是 $(x_k+1, y_k-1)$：

圆方程：$f(x, y) = x^2 + y^2 - R^2 = 0$

$$p_k = f(x_k+1, y_k-\tfrac{1}{2})$$

增量递推：
$$p_{k+1} = \begin{cases} p_k + 2x_{k+1} + 1 & \text{if } p_k < 0 \\ p_k + 2x_{k+1} + 1 - 2y_{k+1} & \text{if } p_k \geq 0 \end{cases}$$

初始值：$p_0 = \frac{5}{4} - R$（实践中用整数 $p_0 = 1 - R$）

### 2.5 DDA vs Bresenham 对比

| 维度 | DDA | Bresenham |
| :--- | :--- | :--- |
| **运算类型** | 浮点加减 + 取整 | 纯整数加减 |
| **精度** | 有累积误差 | 无累积误差（精确） |
| **速度** | 较慢（浮点） | 快（整数） |
| **硬件实现** | 不友好 | 极其友好（无需 FPU） |
| **适用场景** | 教学理解 | 实际工程/硬件 |

### 2.6 工程问题

- **线宽 (Line Width)**：多像素宽度的线如何光栅化
- **端点形状 (Cap)**：平头 (Flat)、圆头 (Round)、方头 (Square)
- **连接处 (Join)**：斜接 (Miter)、圆接 (Round)、平接 (Bevel)

## 3. 曲线绘制

### 3.1 两种基本思路

1. **参数方程 + 线段逼近 (Parametric + Line Approximation)**：$x=f(t), y=f(t)$，取多个 t 值，画直线段逼近
2. **中点细分法**：递归细分直到曲线段足够平直

### 3.2 贝塞尔曲线 (Bézier Curve)

#### 数学定义（Bernstein 多项式）

n 次贝塞尔曲线：
$$\mathbf{B}(t) = \sum_{i=0}^{n} \binom{n}{i} (1-t)^{n-i} t^i \, \mathbf{P}_i, \quad t \in [0,1]$$

其中 $\mathbf{P}_i$ 为控制点，$\binom{n}{i}(1-t)^{n-i}t^i$ 为 Bernstein 基函数。

**性质**：
- 曲线过首尾控制点：$\mathbf{B}(0)=\mathbf{P}_0$, $\mathbf{B}(1)=\mathbf{P}_n$
- 曲线在控制多边形凸包内（**凸包性 (Convex Hull Property)**）
- **仿射不变性 (Affine Invariance)**：对控制点做仿射变换 = 对曲线做同样的仿射变换

#### de Casteljau 算法

通过递归线性插值计算曲线上任意点，无需直接计算多项式：

$$P_i^{(0)} = P_i$$
$$P_i^{(k)} = (1-t)P_i^{(k-1)} + t P_{i+1}^{(k-1)}$$

$P_0^{(n)}$ 即为曲线上 $t$ 处的点。**这是中点细分法的数学基础。**

### 3.3 中点细分法 (Midpoint Subdivision)

**核心思想**：分而治之 — 在 $t=0.5$ 处将曲线一分为二，递归细分直到每一段足够平坦（可用直线段近似）。

#### 三次贝塞尔曲线细分步骤

给定控制点 $\{P_0, P_1, P_2, P_3\}$：

**第一级中点**：
- $P_{0,1}$ = $(P_0 + P_1)/2$
- $P_{1,1}$ = $(P_1 + P_2)/2$
- $P_{2,1}$ = $(P_2 + P_3)/2$

**第二级中点**：
- $P_{0,2}$ = $(P_{0,1} + P_{1,1})/2$
- $P_{1,2}$ = $(P_{1,1} + P_{2,1})/2$

**第三级中点（曲线中点 $t=0.5$）**：
- $P_{0,3}$ = $(P_{0,2} + P_{1,2})/2$

**结果**：
- 左子曲线：$\{P_0, P_{0,1}, P_{0,2}, P_{0,3}\}$
- 右子曲线：$\{P_{0,3}, P_{1,2}, P_{2,1}, P_3\}$

**收敛性**：细分 $m$ 次后，控制多边形以 $O(1/2^m)$ 速率收敛于原曲线。

### 3.4 B-Spline（B样条）

> **考点：** B-Spline 是 Bézier 的推广，解决了 Bézier 曲线"牵一发而动全身"的问题。

#### 动机：Bézier 曲线的局限

Bézier 曲线的两大痛点：
1. **全局性**：移动任意一个控制点 → 整条曲线变形（无局部控制）
2. **次数受限**：高阶 Bézier 曲线（如 $$n>5$$）数值不稳定，低阶又无法表达复杂形状

**B-Spline 的解决方案**：将控制点分段影响曲线，每段只受附近几个控制点支配 → **局部控制 (Local Control)**。

#### 数学定义

B-Spline 由三要素定义：
- **控制点** $\mathbf{P}_0, \ldots, \mathbf{P}_n$
- **节点向量 (Knot Vector)** $T = [t_0, t_1, \ldots, t_{n+k+1}]$（非递减序列）
- **阶数 (Order)** $k$（次数 = $k-1$）

$$C(t) = \sum_{i=0}^{n} \mathbf{P}_i N_{i,k}(t), \quad t \in [t_{k-1}, t_{n+1}]$$

其中 **B-Spline 基函数** $N_{i,k}(t)$ 由 **Cox-de Boor 递推公式** 定义：

$$N_{i,1}(t) = \begin{cases} 1 & \text{if } t_i \leq t < t_{i+1} \\ 0 & \text{otherwise} \end{cases}$$

$$N_{i,k}(t) = \frac{t - t_i}{t_{i+k-1} - t_i} N_{i,k-1}(t) + \frac{t_{i+k} - t}{t_{i+k} - t_{i+1}} N_{i+1,k-1}(t)$$

#### 均匀 (Uniform) vs 非均匀 (Non-Uniform) 节点

| 类型 | 节点分布 | 特点 |
|:---|:---|:---|
| **均匀 B-Spline** | 等距节点（如 $0,1,2,3,\ldots$） | 简单，但端点处无插值 |
| **非均匀 B-Spline** | 不等距 | 可控制端点插值行为 |
| **Clamped / Open Uniform** | 两端节点重复 $k$ 次 | 曲线通过首尾控制点（最常用） |

#### B-Spline 的关键性质

| 性质 | 说明 |
|:---|:---|
| **局部控制 (Local Control)** | 移动 $\mathbf{P}_i$ 只影响区间 $[t_i, t_{i+k}]$ 内的曲线 → 比 Bézier 更灵活 |
| **连续性** | $C^{k-d-1}$ 连续（$d$ 为节点重复度），可自然表达任意阶光滑 |
| **凸包性 (Convex Hull Property)** | 每段曲线落在对应 $k$ 个控制点的凸包内 → 更强的局部约束 |
| **变差缩减性** | 曲线与任意直线求交次数 ≤ 控制多边形与该直线的求交次数 |
| **仿射不变性** | 同 Bézier：变换控制点 = 变换曲线 |
| **包含 Bézier** | 当控制点数 = 阶数，且节点为 Clamped → 退化为 Bézier 曲线 |

#### B-Spline vs Bézier

| 维度 | Bézier | B-Spline |
|:---|:---|:---|
| 控制点数 | = 次数 + 1 | 独立于次数 |
| 局部控制 | 无（移动一个点整条线变） | 有（只影响局部） |
| 自由度 | 低 | 高（节点 + 控制点） |
| 端点插值 | 自动插值 | 需 Clamped 节点 |
| 标准 | 字体 (TrueType) | CAD 建模 |

### 3.5 NURBS（非均匀有理B样条）

> **考点：** NURBS 的全称和核心优势（精确表达圆锥曲线）。

#### 定义

**NURBS** = **Non-Uniform Rational B-Spline**（非均匀有理B样条）。

从 B-Spline 推广：为每个控制点分配一个**权重 (Weight)** $w_i$，在齐次坐标 $\mathbb{R}^4$ 中定义 B-Spline，再投影回 $\mathbb{R}^3$：

$$C(t) = \frac{\sum_{i=0}^{n} w_i \mathbf{P}_i N_{i,k}(t)}{\sum_{i=0}^{n} w_i N_{i,k}(t)}$$

- 当所有 $w_i = 1$ → 退化为普通 B-Spline
- $w_i$ 越大 → 曲线越"靠近"该控制点
- $w_i = 0$ → 该控制点失去影响力（无穷远点）

#### 为什么需要 NURBS？

**核心动机**：B-Spline（以及 Bézier）是**多项式曲线**，无法精确表达**圆、椭圆、双曲线**等圆锥曲线——这些在工程制图中无处不在。

**NURBS 是唯一能精确表达所有圆锥曲线的多项式/有理曲线族。**

这正是它成为工业标准（**STEP、IGES、OpenGL `gluNurbsCurve`**）的原因。

#### 各组成部分的含义

| 缩写 | 全称 | 作用 |
|:---|:---|:---|
| **N**on-Uniform | 节点间距不相等 | 控制端点插值 + 连续性可调 |
| **R**ational | 使用齐次权重 | **精确表达圆锥曲线** |
| **B**-Spline | B样条基函数 | 局部控制 + 高阶连续 |

#### 应用场景

- **CAD/CAM**：所有主流建模软件（SolidWorks、CATIA、Rhino）的标准几何表示
- **字体**：PostScript Type 1 和 OpenType 中的轮廓
- **动画路径**：平滑的相机和角色运动轨迹
- **数据可视化**：任意精度的平滑曲线拟合

## 4. 多边形填充

### 4.1 核心思想：降维

将二维多边形填充 → 逐行的一维区间填充：
- 用水平扫描线从上到下扫描多边形
- 每条扫描线与多边形边界求交点
- 交点排序配对，填充配对区间

### 4.2 奇偶规则 (Odd-Even Rule / Odd-Parity Rule)

> **考点：** 基本概念，常考判断点是否在多边形内。

从待测点向任意方向发射射线：
- **奇数次**穿过多边形边界 → 在内部
- **偶数次**穿过多边形边界 → 在外部

### 4.3 奇偶规则 vs 非零环绕规则 (Non-Zero Winding Rule)

| 规则 | 判断依据 | 适用场景 |
| :--- | :--- | :--- |
| **奇偶规则** | 穿边次数 mod 2 | 简单填充，如 SVG（可缩放矢量图形）`fill-rule="evenodd"` |
| **非零环绕** | 环绕数 (winding number) $$\neq 0$$ | 自交多边形，如 SVG `fill-rule="nonzero"` |

**非零环绕规则**：从待测点发射线，边从左到右穿 +1，从右到左穿 -1，累加值 $$\neq 0$$ 则在内部。

### 4.4 扫描线填充算法

#### 核心数据结构

**边表 (ET, Edge Table)**：
- 按边的最小 y 值索引
- 每条边存储：`ymax`（最大 y）、`x`（较低端点的 x）、`dx`（斜率倒数 $1/m$）

**活化边表 (AET, Active Edge Table)**：
- 存储与当前扫描线相交的所有边
- 按 x 值从小到大排序

#### 算法流程

```
初始化：AET = ∅
y = 最小y值

while y <= 最大y值:
    ① 将ET[y]中所有边加入AET
    ② 从AET中删除 ymax == y 的边
    ③ 对AET按x值排序
    ④ 两两配对，填充区间 [x₀,x₁], [x₂,x₃], ...
    ⑤ 更新AET中每条边的 x = x + dx
    ⑥ y = y + 1
```

#### 关键操作物理含义

| 操作 | 物理含义 |
| :--- | :--- |
| ① 加入边 | 扫描线到达边的起始位置 |
| ② 删除边 | 扫描线越过边的终止位置 |
| ③ 排序 | 从左到右排列交点 |
| ④ 配对填充 | 奇偶规则：进入→离开→进入→离开 |
| ⑤ 更新 x | 沿边方向前进一个像素行 |

#### 边界处理要点

**水平边 (Horizontal Edge)**：忽略（不加入 ET），由相邻非水平边隐含表示。

**顶点共享问题**（最关键）：
- 扫描线恰好经过顶点时产生歧义
- 解决方案：边只包含**下端点**，不包含**上端点**（**下闭上开 (Bottom-closed Top-open)**）

```
情况A：局部极值点 → 计为 1 个交点
情况B：普通共享顶点 → 通过"下闭上开"规则避免重复计数
```

**像素取整规则**：
- 左边界交点：`ceil(x)` 向上取整
- 右边界交点：`floor(x)` 向下取整

#### 算法复杂度

| 阶段 | 复杂度 | 说明 |
| :--- | :--- | :--- |
| 建表 | $O(n)$ | n 为顶点数 |
| 排序 | $O(m \log m)$ | m 为活跃边数 |
| 填充 | $O(h \times k)$ | h 为扫描行数，k 为平均交点数 |
| **总体** | $O(n \times h)$ | 最坏情况 |

空间复杂度：$O(n)$，存储 ET + AET。

#### 算法变体

| 变体 | 特点 |
| :--- | :--- |
| **有序边表法** | ET+AET，最常用 |
| **栅栏填充法** | 不维护 AET，每行重算，简单但慢 |
| **边界标志法** | 先标记边界再扫描填充，适合硬件 |

## 5. 区域填充

### 5.1 种子填充 (Seed Fill / Flood Fill)

从种子点开始，递归扩展到相邻像素，直到遇到边界。

| 规则 | 扩展方向 | 特点 |
| :--- | :--- | :--- |
| **4连通 (4-connected)** | 上下左右 | 快，可能漏填对角线区域 |
| **8连通 (8-connected)** | 上下左右+对角线 | 更完整，可能穿过4连通边界 |

### 5.2 扫描线种子填充

改进：将逐点递归 → 逐段填充，减少递归深度，避免栈溢出。

### 5.3 边界填充 (Boundary Fill)

从种子点向外扩展，遇到**指定边界色**停止。

### 5.4 算法选择指南

| 场景 | 推荐算法 | 原因 |
| :--- | :--- | :--- |
| CAD 绘制多边形 | 扫描线填充 | 精确、可控、无递归 |
| 图像编辑（魔术棒） | 泛洪填充 | 按颜色匹配自然 |
| 小游戏/像素画 | 4连通种子 | 实现简单 |
| 大纹理区域 | 扫描线种子 | 内存友好 |
| 消除锯齿 | 8连通种子 | 更自然的边界 |

## 6. 反走样 (Anti-aliasing)

### 6.1 走样的信号处理视角

> **考点：** 老师可能从信号处理角度出概念题。

**走样 (Aliasing)** 的根本原因：**离散采样违反奈奎斯特采样定理**。

**奈奎斯特定理**：要完美重建信号，采样频率必须 $$\geq$$ 信号最高频率的 2 倍。

光栅化中每个像素是一个采样点，采样频率 = 像素密度。场景中的高频信息（尖锐边缘、细线、高对比度纹理）超过了奈奎斯特频率，导致走样。

走样表现：锯齿边缘、摩尔纹 (Moiré)、细节丢失 (Detail Loss)、闪烁 (Flickering)、时间走样 (Temporal Aliasing)。

### 6.2 反走样技术对比

| 技术 | 核心思想 | 处理阶段 | 性能 | 质量 |
| :--- | :--- | :--- | :--- | :--- |
| **SSAA** | 高分辨率渲染 → 降采样 | 渲染时 | 极低（$$n\times$$像素） | 最高 |
| **MSAA** | 仅边缘超采样，着色算一次 | 渲染时 | 中 | 高 |
| **FXAA** | 后处理边缘检测+模糊 | 后处理 | 高 | 中 |
| **MLAA** | 图像模式识别锯齿形状 | 后处理 | 高 | 中 |
| **TAA** | 多帧抖动采样累积 | 渲染时+后处理 | 中高 | 高（动态） |
| **距离场** | 存储有符号距离值精确计算覆盖 | 渲染时 | 中 | 高（文字） |
| **DLSS/FSR** | AI 超分辨率重建 | AI 加速 | 中（需 Tensor Core） | 极高 |

### 6.3 各技术详解

**SSAA (Super-Sampling Anti-Aliasing)**：
- 每个像素取多个子样本点
- 计算子样本的平均颜色
- 本质：用计算量换图像质量
- 缺点：$$n\times$$ 分辨率 = $$n\times$$ 像素着色成本

**MSAA (Multi-Sample Anti-Aliasing)**：
- SSAA 的优化版：深度/覆盖测试多次，但**着色只执行一次**
- 对每个像素维护多个样本的覆盖信息
- 是 GPU 硬件最广泛支持的标准抗锯齿

**FXAA (Fast Approximate Anti-Aliasing)**：
- 后处理：对渲染完成的图像做边缘检测和平滑
- 沿边缘方向进行模糊混合
- 优点：不依赖硬件 MSAA 支持，速度快
- 缺点：可能导致纹理模糊

**TAA (Temporal Anti-Aliasing)**：
- 每帧采样位置略微偏移（jitter）
- 融合历史帧信息来平滑边缘
- 优点：动态场景效果好，同时能处理着色走样
- 缺点：静止时可能产生鬼影 (ghosting)

**DLSS / FSR / XeSS**：
- 用深度学习/上采样算法从低分辨率重建高分辨率
- 同时实现反走样和性能提升
- 现代游戏的标配方案

## 常见考点

1. **推导 Bresenham 算法的决策参数增量公式** → 计算题
2. **给定两端点，手算 Bresenham 每一步的 p 值和 y 值** → 填表题
3. **Bresenham 画圆算法为什么只需计算八分圆？** → 利用八对称性
4. **为什么 DDA 有累积误差而 Bresenham 没有？** → DDA 用浮点数加 m，有舍入误差累积
5. **扫描线填充中 AET 和 ET 的作用分别是什么？** → ET 存储所有非水平边（按 y_min 索引），AET 存储与当前扫描线相交的边
6. **顶点共享问题如何解决？** → "下闭上开"规则，边只包含下端点
7. **走样的信号处理原理？** → 采样频率 < $$2\times$$ 信号最高频率（违反奈奎斯特定理）
8. **SSAA vs MSAA 的本质区别？** → SSAA 每个子样本都着色，MSAA 只着色一次
9. **de Casteljau 算法和贝塞尔曲线的关系？** → de Casteljau 是贝塞尔曲线的递归求值/细分算法
10. **B-Spline 相比 Bézier 的核心优势？** → 局部控制：移动一个控制点只影响局部曲线段
11. **NURBS 全称是什么？为什么能精确表达圆？** → Non-Uniform Rational B-Spline；通过有理权重在齐次坐标下表达圆锥曲线
