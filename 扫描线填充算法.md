# 扫描线填充算法

## 一、算法思想

扫描线填充算法的核心思想是**降维处理**：将二维多边形填充问题转化为一维区间填充问题。

### 基本思路

1. **水平扫描**：用水平扫描线从上到下（或从下到上）扫描多边形
2. **求交计算**：计算每条扫描线与多边形边界的交点
3. **区间填充**：将交点排序配对，填充配对区间内的像素

**为什么有效？**

- 任何水平线与凸多边形的交点是**连续区间**
- 对于凹多边形，交点可能多个，但**成对出现**
- 扫描线逐行处理，每行只需处理一维区间

---

## 二、核心数据结构

### 1. 边表（ET - Edge Table）

- **作用**：存储所有非水平边，按边的**最小y值**索引
- **节点内容**：
  - `ymax`：边的最大y值（扫描终止线）
  - `x`：边的**较低端点**的x坐标
  - `dx`：斜率倒数 `(x₂-x₁)/(y₂-y₁)`

### 2. 活化边表（AET - Active Edge Table）

- **作用**：存储当前扫描线相交的所有边
- **动态更新**：随着扫描线移动，边的x值增量更新
- **排序要求**：始终按x值从小到大排列

```
边表结构示例：
ET[y_min] = [边1, 边2, ...]  // y_min为边的较低端点y值
AET = [边a, 边b, ...]       // 当前活跃边，按x排序
```

---

## 三、算法流程

### 主循环（从下到上扫描）

```
初始化：AET = ∅
y = 最小y值

while y ≤ 最大y值:
    ① 将ET[y]中所有边加入AET
    ② 从AET中删除ymax == y的边（扫描线越过边）
    ③ 对AET按x值排序
    ④ 两两配对，填充区间 [x₀,x₁], [x₂,x₃], ...
    ⑤ 更新AET中每条边的x = x + dx
    ⑥ y = y + 1
```

### 关键操作的物理含义

| 操作       | 物理含义                  |
| ---------- | ------------------------- |
| ① 加入边   | 扫描线到达边的起始位置    |
| ② 删除边   | 扫描线离开边的终止位置    |
| ③ 排序     | 从左到右遍历交点          |
| ④ 配对填充 | 进入/离开多边形的边界配对 |
| ⑤ 更新x    | 沿边的方向前进一个像素行  |

## 四、边界处理要点

### 1. 水平边处理

- **忽略水平边**，避免产生无限多交点
- 水平边由相邻非水平边隐含表示

### 2. 顶点共享问题（最关键！）

当扫描线经过顶点时，会出现**共享顶点**的歧义：

```
情况A：局部极值点（尖点）
  ┌─┐    → 扫描线刚好经过顶点，属于多边形的"转折"
  └─┘       计为1个交点（边计1次）

情况B：非极值点（普通顶点）  
  ─┐      → 扫描线穿过顶点，属于"穿过"边界
   │         计为2个交点（或0次，取决于实现）
  ─┘
```

**解决方案**：

- **方法1**：检查相邻顶点y值
  - 若 `(y₁ < y₂ and y₁ < y₀)` 或 `(y₁ > y₂ and y₁ > y₀)` → 极值点
  - 否则 → 普通点

- **方法2**：边缩短（边中包含ymax但不包含ymin）
  - 每条边只包含**下端点**，不包含**上端点**
  - 确保每条边在ymax处被删除，在ymin处被加入

### 3. 像素取整规则

| 交点位置   | 取整规则                       |
| ---------- | ------------------------------ |
| 左边界交点 | `ceil(x)` 向上取整             |
| 右边界交点 | `floor(x)` 向下取整            |
| 整数交点   | 直接使用，但需配合顶点处理规则 |

## 五、配对原理

### 为什么两两配对有效？

基于**Jordan曲线定理**：

- 从多边形外部沿水平线向右移动
- 每遇到一个交点，就**切换一次**内外状态
- 奇数次交点 → 进入多边形内部
- 偶数次交点 → 离开多边形外部

```
配对规则：
交点排序后: [x₀, x₁, x₂, x₃, x₄, x₅]
填充区间:   [x₀, x₁], [x₂, x₃], [x₄, x₅]
                ↑         ↑         ↑
            进入-离开  进入-离开  进入-离开
```

### 配对时的特殊情况

**切线情况**（与顶点重合）：

- 若扫描线恰好经过顶点，按顶点处理规则**调整边计数**
- 确保交点数仍为偶数

## 六、增量计算的优势

### 核心思想：避免重复计算

对于边 `(x₁,y₁) → (x₂,y₂)`：

- **斜率倒数**：`dx = (x₂-x₁)/(y₂-y₁)`
- **当前交点**：当前扫描线y处的x坐标
- **下一行**：`x_next = x_current + dx`

**为什么不用斜率而用倒数？**

- 扫描线按y递增，x是y的函数
- 直接使用 `x = x + dx` 避免除法运算
- 整数y间隔为1，增量计算高效

## 七、算法变体

### 1. 有序边表法（最常用）

- ET按y排序，AET按x排序
- 每条扫描线只需处理**新增边**和**删除边**
- 时间复杂度 O(n log n)

### 2. 栅栏填充法

- 不维护AET，每行重新计算交点
- 实现简单，但效率较低
- 适用于小规模或静态图形

### 3. 边界标志法

- 先标记所有边界像素
- 再扫描行，奇偶填充
- 适合硬件实现

---

## 八、常见错误及解决

### 错误1：填充出现断线

**原因**：交点配对错误或浮点精度问题

**解决**：

- 使用 `ceil/floor` 而不是 `int` 强制转换
- 处理顶点时确保交点为偶数

### 错误2：凹多边形填充错误

**原因**：未正确处理共享顶点

**解决**：

- 严格执行"上端点不包含"规则
- 或使用奇偶规则判断

### 错误3：自交多边形失败

**原因**：标准扫描线算法假设简单多边形

**解决**：

- 使用**奇偶规则**（Odd-Even Rule）而非环绕规则
- 或者先分解为简单多边形

## 九、算法复杂度

| 阶段 | 时间复杂度 | 说明                               |
| ---- | ---------- | ---------------------------------- |
| 建表 | O(n)       | n为顶点数                          |
| 排序 | O(m log m) | m为活跃边数，最多n条               |
| 填充 | O(h × k)   | h为高度（扫描行数），k为平均交点数 |
| 总体 | O(n × h)   | 最坏情况                           |

**空间复杂度**：O(n)，存储边表和活性边

---

## 十、实际应用注意事项

1. **坐标系**：屏幕坐标y轴向下，算法用数学坐标y轴向上，需转换
2. **浮点精度**：使用 `double` 而非 `float`，避免累积误差
3. **边界像素**：根据图形API的约定（包含左边界还是右边界）
4. **抗锯齿**：扫描线算法天然可扩展为**反走样**版本
5. **硬件加速**：现代GPU使用**光栅化**，本质是并行扫描线

---

**总结**：扫描线填充的本质是将**二维区域填充**分解为**一维区间填充**，利用增量更新和有序结构，实现高效的多边形光栅化。核心难点在于**边界顶点的正确处理**和**交点配对逻辑**。
