# Cohen-Sutherland Line Clipping

**把屏幕空间分成九宫格，用四位二进制码标记端点位置；先通过位运算快速接受或拒绝线段，无法判定时再循环求交，逐步逼近窗口边界。**

### 第一步：区域编码（Region Codes）

算法把屏幕窗口的**九个区域**用**4位二进制码**标记。对于一条线段的两个端点，分别计算它们的编码。

- **上 (Top)**：第1位（`y > y_max`）
- **下 (Bottom)**：第2位（`y < y_min`）
- **右 (Right)**：第3位（`x > x_max`）
- **左 (Left)**：第4位（`x < x_min`）

比如，窗口左上角区域的编码就是 `1001`（上 + 左）。

------

### 第二步：快速判定（两种极端情况）

对于每条线段，用两个端点的编码进行逻辑判断：

1. **完全可见（Trivial Accept）**：如果两个端点编码**都是 `0000`**，说明线段完全在窗口内，直接保留。
2. **完全不可见（Trivial Reject）**：如果两个端点编码**按位与（AND）结果不为 0**，说明线段在窗口的同一外侧（例如都在窗口上方），直接丢弃。

> **关键优化**：这一步全是**整数位运算**，极其快速，能高效处理掉绝大部分简单线段。

------

### 第三步：求交裁剪

如果线段不能通过上述两种快速判定，就需要计算它与窗口边界的交点，然后**用交点替换掉外侧的端点**，重复上述测试。

**求交顺序的技巧**：通常按照 **上 → 下 → 右 → 左** 的顺序（固定顺序）进行裁剪。每次计算一个交点，替换端点编码，然后重新进入第二步测试。

> **注意**：该算法一次只裁剪掉一个边界。对于横跨多个边界的线段，可能需要多次迭代（最多4次）才能完全裁剪干净。
