## 1. 全局光照概述

**局部光照**（Ch.07）：只考虑光源→表面→眼睛的直接路径。

**全局光照 (GI)**：考虑光的所有路径——包括反射、折射、间接弹射。是让画面从"一眼假"进化到"照片级"的核心技术。

### 离线 vs 实时 GI

| 维度 | 离线 GI | 实时 GI |
| :--- | :--- | :--- |
| 代表技术 | Path Tracing, Photon Mapping | Lightmap, Probes, SSGI, Lumen |
| 计算时间 | 分钟～小时/帧 | <33ms/帧 |
| 质量 | 物理精确 | 近似 |
| 场景变动 | 静态 | 可动态 |
| 应用 | 电影、建筑可视化 | 游戏、VR |

## 2. 渲染方程

> ⚠️ 全局光照的理论基石（Kajiya 1986），必须理解每一项的物理含义。

### 2.1 标准形式

$$L_o(p,\omega_o) = L_e(p,\omega_o) + \int_{\Omega^+} L_i(p,\omega_i) \, f_r(p,\omega_i,\omega_o) \, (\mathbf{n} \cdot \omega_i) \, d\omega_i$$

### 2.2 统一形式（Kajiya 原始论文）

$$I(x, x') = g(x, x') \left[ \epsilon(x, x') + \int_S \rho(x, x', x'') I(x', x'') \, dx'' \right]$$

| 符号 | 数学含义 | 物理直觉 |
| :--- | :--- | :--- |
| $I(x, x')$ | 从 $x'$ 到 $x$ 的传输强度 | 两点之间的"亮度" |
| $g(x, x')$ | $1/r^2$ 或 0 | 距离衰减 + 可见性 |
| $\epsilon(x, x')$ | 自发光强度 | 光源本身发光 |
| $\rho(x, x', x'')$ | 三点散射函数 | 表面如何将 $x''$ 来的光反射到 $x$ |

### 2.3 算子形式与 Neumann 级数

写成积分算子形式：$\mathcal{L} = L_e + \mathcal{T}\mathcal{L}$

其中 $\mathcal{T}$ 是光传输算子。这是 **Fredholm 第二类积分方程**。

解为 Neumann 级数：
$$\mathcal{L} = \sum_{k=0}^{\infty} \mathcal{T}^k L_e$$

**物理含义**：
- $k=0$：直接光照（光源→眼睛）
- $k=1$：一次弹射（光源→表面→眼睛）
- $k=2$：两次弹射（间接光照）
- $k \to \infty$：完整的全局光照

> 这个级数展开是路径追踪和双向路径追踪的理论基础。

### 2.4 与辐射度方程的对比

| 特性 | 辐射度方程 | 渲染方程 |
| :--- | :--- | :--- |
| 表面假设 | 仅**理想漫反射** | 任意 BRDF |
| 角度依赖 | 无 | 有（完整方向信息） |
| 可见性 | 需额外计算 | 由 $g$ 项统一处理 |
| 适用场景 | 室内漫反射主导 | 全局光照、镜面、折射、焦散 |

### 2.5 渲染方程支持的光学现象

| 现象 | 方程中的体现 |
| :--- | :--- |
| 阴影 | $g(x,x')=0$（被遮挡） |
| 漫反射 | $\rho$ 为常数 |
| 镜面反射/折射 | $\rho$ 为 Dirac delta 函数形式 |
| 颜色溢出 (Color Bleeding) | 多次反射项 |
| 焦散 (Caustics) | 光通过折射表面聚焦 |
| 环境光照 | 来自背景表面 $S_0$ |

## 3. Whitted 递归光线追踪

> Whitted (1980) — 递归光线追踪的奠基性工作。

### 3.1 核心创新

光线从视点出发，在交点处生成：
- **反射光线**（沿 $\mathbf{R}$ 方向）
- **折射光线**（沿 $\mathbf{P}$ 方向，Snell 定律）
- **阴影光线**（指向每个光源）

递归处理 → 形成**光线树 (Ray Tree)**。

### 3.2 Whitted 光照公式

$$I = I_a + k_d \sum_{j=1}^{l_s} (\mathbf{N} \cdot \mathbf{L}_j) + k_s S + k_t T$$

- $S$：沿反射方向 $\mathbf{R}$ 入射的光强（来自其他物体的反射）
- $T$：沿折射方向 $\mathbf{P}$ 入射的光强（透射光）
- $k_s, k_t$：镜面反射和透射系数

> $S$ 和 $T$ 递归计算，体现全局光照。

### 3.3 论文性能数据（历史参考）
- 480×640 分辨率，VAX-11/780
- 渲染时间：44～122 分钟/帧
- 75%～95% 时间花在相交计算

## 4. 光线追踪核心机制

### 4.1 光线方程

$$\mathbf{R}(t) = \mathbf{E} + t(\mathbf{P} - \mathbf{E}), \quad t \in [1, +\infty)$$

- $\mathbf{E}$：视点，$t=0$ 在眼睛处
- $\mathbf{P}$：像素中心在视平面上的位置，$t=1$ 在像素处

### 4.2 光线类型

| 类型 | 发射起点 | 方向 | 用途 |
| :--- | :--- | :--- | :--- |
| **Eye Ray** | 眼睛 | 穿过像素 | 初始采样 |
| **Shadow Ray** | 交点 | 指向光源 | 判断遮挡 |
| **Reflection Ray** | 交点 | 反射方向 $\mathbf{R}$ | 镜面反射 |
| **Refraction Ray** | 交点 | 折射方向 $\mathbf{P}$ | 透明/折射 |

### 4.3 反射方向计算

入射方向 $\mathbf{V}$（指向视点），法线 $\mathbf{N}$（指向入射侧）：

$$\mathbf{V}' = \frac{\mathbf{V}}{|\mathbf{V} \cdot \mathbf{N}|}, \quad \mathbf{R} = \mathbf{V}' + 2\mathbf{N}$$

### 4.4 折射方向（Snell 定律）

$$n_1 \sin\theta_1 = n_2 \sin\theta_2$$

$$\mathbf{P} = k_f(\mathbf{N} + \mathbf{V}') - \mathbf{N}, \quad k_f = \left(k_n^2 |\mathbf{V}'|^2 - |\mathbf{V}'+\mathbf{N}|^2\right)^{-1/2}$$

$k_n$：相对折射率（入射介质/折射介质）。

**全内反射**：根号内为负时发生，$T=0$。

### 4.5 光线-三角形求交（Möller-Trumbore）

$$\mathbf{O} + t\mathbf{D} = (1-u-v)\mathbf{V}_0 + u\mathbf{V}_1 + v\mathbf{V}_2$$

解出 $t, u, v$，判断 $t>0$ 且 $u,v \geq 0$ 且 $u+v \leq 1$。

### 4.6 光线-AABB 相交（Slabs 测试）

$$t_{\min} = \max\left(\frac{x_{\min}-O_x}{D_x}, \frac{y_{\min}-O_y}{D_y}, \frac{z_{\min}-O_z}{D_z}\right)$$
$$t_{\max} = \min\left(\frac{x_{\max}-O_x}{D_x}, \frac{y_{\max}-O_y}{D_y}, \frac{z_{\max}-O_z}{D_z}\right)$$

有效交点条件：$t_{\min} < t_{\max}$ 且 $t_{\max} > 0$。

## 5. 加速结构

### 5.1 为什么需要？

朴素光线追踪：每条光线 × 所有图元 → $O(rays \times primitives)$，不可行。

**目标**：快速剔除不可能相交的物体，利用空间相干性。

### 5.2 加速结构分类

| 类别 | 代表 |
| :--- | :--- |
| **基于物体 (Object-based)** | 包围体 (BV)、BVH |
| **基于空间 (Space-based)** | 均匀网格、八叉树、KD-Tree、BSP树 |

### 5.3 包围体 (Bounding Volume)

**思想**：用简单几何体（AABB、球体）包围复杂物体 → 先测试光线是否击中包围体 → 未击中则跳过内部所有图元。

### 5.4 BVH (Bounding Volume Hierarchy)

**树形结构**：
- 内部节点：包围体 + 子节点指针
- 叶子节点：实际图元列表

**构建**（自顶向下）：递归二分图元 → 计算包围盒 → SAH 优化划分。

**遍历**：
```
Intersect(node, ray):
    if ray misses node.bbox → return
    if node is leaf → test all primitives, update min_t
    else → Intersect(left), Intersect(right)
```

**优化**：**按距离排序子节点**，先处理近的（Early Termination）。

**复杂度**：构建 $O(N \log N)$，查询平均 $O(\log N)$。

### 5.5 均匀网格 (Uniform Grid)

- 空间划分为相同大小的体素 (Voxel)
- 光线用 3D DDA 沿方向逐格前进
- 优点：简单；缺点：分辨率难选（"teapot in stadium"问题）

### 5.6 八叉树 (Octree)

- 递归 8 等分立方体空间
- 自适应：稀疏区域少划分，密集区域多划分

### 5.7 KD-Tree

- 每次用一个**轴对齐平面**二分空间
- 交替或 SAH 选择划分轴和位置
- 内部节点不存图元，叶子存图元列表
- 优点：高效；缺点：构建慢、动态场景差

### 5.8 BSP 树

- 与 KD-Tree 类似，但划分平面**不限于轴对齐**
- 更灵活但构建和遍历开销更大

### 5.9 SAH (Surface Area Heuristic)

> 选择划分位置的启发式方法。

$$C(V) = C_T + \frac{SA(V_L)}{SA(V)} C(V_L) + \frac{SA(V_R)}{SA(V)} C(V_R)$$

- $C_T$：遍历节点的代价
- $SA(V_L)/SA(V)$：光线击中左子节点的概率（正比于表面积）
- 选择使 $C(V)$ 最小的划分

### 5.10 加速结构对比总表

| 结构 | 构建 | 查询 | 内存 | 动态 | 实现 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 均匀网格 | 低 | 中 | 高 | 中 | 简单 |
| 八叉树 | 中 | 中高 | 中 | 中 | 中 |
| KD-Tree | 高 | 高 | 中 | 差 | 高 |
| **BVH** | 中 | 高 | 低 | **好** | 中 |
| BSP 树 | 高 | 中高 | 中 | 差 | 高 |

> **BVH 是现代 GPU 光线追踪 (RTX, OptiX, Vulkan RT) 的标准选择。**

## 6. 分布式光线追踪：高级效果

### 6.1 抗锯齿 (Anti-Aliasing)

每个像素发射多条光线（4/16/256），采样点在像素内随机抖动 (Jitter)，取平均。锯齿 → **噪声**（视觉上更可接受）。

### 6.2 软阴影 (Soft Shadows)

光源不是点而**有面积**（面光源）→ 半影 (Penumbra)。从交点向光源表面多个点发射 Shadow Ray，统计可见比例。

### 6.3 景深 (Depth of Field)

真实相机有光圈（非针孔）→ 焦外模糊。在镜头表面随机采样多个点，光线从不同位置穿过同一像素。

## 7. 路径追踪 (Path Tracing)

> Whitted 光线追踪在漫反射场景中力不从心。路径追踪是更通用的蒙特卡洛解。

### 7.1 核心思想

每次碰撞只选**一条后继路径**（而非光线树的多条分支），通过大量路径的统计平均逼近渲染方程。

### 7.2 路径追踪伪代码

```cpp
Color pathTrace(Ray ray):
    Color throughput = (1,1,1)
    Color radiance = (0,0,0)
    
    for bounce = 0 to maxDepth:
        hit = intersect(ray)
        if !hit: break
        
        // 直接光照（Next Event Estimation）
        radiance += throughput * sampleDirectLighting(hit)
        
        // 采样 BSDF 生成下一跳方向
        wi, bsdfVal, pdf = sampleBSDF(hit, -ray.dir)
        throughput *= bsdfVal * abs(dot(hit.n, wi)) / pdf
        
        // Russian Roulette 终止
        if bounce > 3:
            q = max(0.05, 1 - maxComponent(throughput))
            if random() < q: break
            throughput /= (1 - q)
        
        ray = Ray(hit.p, wi)
    
    return radiance
```

### 7.3 路径积分形式（理论基础）

长度为 $k$ 的路径 $\bar{x}=x_0 x_1 \cdots x_k$ 的贡献：

$$I(\bar{x}) = L_e(x_k \to x_{k-1}) \prod_{i=1}^{k-1} f_r(x_{i+1} \to x_i \to x_{i-1}) \prod_{i=1}^{k-1} G(x_i \leftrightarrow x_{i+1})$$

几何项：
$$G(x \leftrightarrow y) = \frac{\cos\theta_x \cos\theta_y}{\|x-y\|^2} V(x,y)$$

$V(x,y) \in \{0,1\}$ 为可见性。像素颜色 = 所有路径贡献的积分。

## 8. 蒙特卡洛积分的数学基础

### 8.1 基本估计量

$$\int f(x) dx \approx \frac{1}{N} \sum_{i=1}^{N} \frac{f(X_i)}{p(X_i)}, \quad X_i \sim p(x)$$

### 8.2 无偏性与一致性

- **无偏性**：$\mathbb{E}[F_N] = \int f(x) dx$（期望等于精确值，推导只需线性期望）
- **一致性**：$\lim_{N \to \infty} F_N = \int f(x) dx$（样本→∞时收敛到精确值）
- **方差**：$\text{Var}(F_N) = \frac{1}{N}\text{Var}\left(\frac{f(X)}{p(X)}\right)$

### 8.3 收敛率

$$\text{MSE} \propto \frac{1}{N}$$

- 要减半噪声 → 需要 **4 倍**样本数
- **维度无关性**：收敛率不随积分维数增加恶化 → Monte Carlo 在高维积分上远优于数值积分

### 8.4 半球均匀采样的 PDF

均匀采样半球：$p(\omega_i) = 1/2\pi$

验证归一化：
$$\int_0^{2\pi}\int_0^{\pi/2} \frac{1}{2\pi} \sin\theta \, d\theta \, d\phi = 1$$

> $\sin\theta$ 来自球坐标 Jacobian，是立体角 $d\omega = \sin\theta \, d\theta \, d\phi$。

## 9. 重要性采样与光源采样

### 9.1 重要性采样原理

**最优 PDF**：$p(x) \propto |f(x)|$（与被积函数成比例），此时方差最小。

### 9.2 光源采样 (Next Event Estimation)

**动机**：半球均匀采样击中光源的概率极低（光源小或远时）。

**核心技巧**：改变积分变量，在光源表面采样而非在半球方向采样。

立体角与面积微元的关系：
$$d\omega_i = \frac{\cos\theta'}{\|x'-x\|^2} dA$$

其中 $\theta'$ 是**光源面元法线**与连线方向的夹角。

面积积分形式的渲染方程：
$$L_o(x,\omega_o) = \int_A L_i \, f_r \, \frac{\cos\theta \cos\theta'}{\|x'-x\|^2} \, dA$$

在光源表面均匀采样（$p(x')=1/A$）：
$$L_o \approx \frac{A}{N}\sum_{k=1}^{N} L_i^{(k)} f_r \frac{\cos\theta \cos\theta'}{\|x_k'-x\|^2} \cdot V(x, x_k')$$

> 采样光源后仍需 Shadow Ray 检测遮挡（$V(x,x_k')$）。

### 9.3 效率对比

设光源面积为 $A_s$，距离为 $r$：
- 半球采样击中概率：$\approx A_s \cos\theta' / 2\pi r^2$
- 光源采样有效概率：$1$（仅遮挡会失效）
- 效率提升：$\approx r^2 / A_s$ 倍

## 10. 高级采样技术

### 10.1 俄罗斯轮盘赌 (Russian Roulette)

> ⚠️ 必考。路径追踪的核心终止策略。

设原始着色结果 $Lo$，存活概率 $P \in (0,1)$：

$$Y = \begin{cases} Lo/P & \text{概率 } P \\ 0 & \text{概率 } 1-P \end{cases}$$

**无偏性证明**：$\mathbb{E}[Y] = P \cdot \frac{Lo}{P} + (1-P) \cdot 0 = Lo$ ✓

**方差**：$\text{Var}(Y) = Lo^2(1/P - 1)$

$P$ 越小 → 方差越大但终止越快 → **引入额外噪声但保证了有限终止且无偏**。

**选择**：$P = \max(0.05, 1 - \max(\text{throughput}))$，高贡献路径更倾向于存活。

### 10.2 多重重要性采样 (MIS)

**问题**：何时采样光源（对直接光照好）vs 采样 BRDF（对间接光照好）？

**MIS 平衡启发式**：组合多个采样策略，权重 $\propto n_i p_i(x)$：

$$F_{MIS} = \sum_{i=1}^{M} \frac{1}{n_i} \sum_{j=1}^{n_i} w_i(X_{i,j}) \frac{f(X_{i,j})}{p_i(X_{i,j})}$$

其中 $w_i(x) = \frac{n_i p_i(x)}{\sum_k n_k p_k(x)}$（平衡启发式）。

**效果**：避免某策略权重极端大的路径 → 方差显著降低。

### 10.3 低差异序列 (Low Discrepancy Sequence)

用确定性序列（Halton, Sobol）代替伪随机数：
- 收敛速度：$O(\log N / N)$ vs 随机 $O(1/\sqrt{N})$
- 更均匀的样本分布（无 clumping）

## 11. 粒子输运视角

> Arvo & Kirk (1990)：渲染方程本质上是线性玻尔兹曼方程的特例。

### 11.1 核心类比

| 概念 | 核工程 | 图形学 |
| :--- | :--- | :--- |
| 粒子 | 中子 | 光子 |
| 输运方程 | 线性玻尔兹曼方程 | 渲染方程 |
| 散射核 | 中子散射截面 | BRDF |
| 源项 | 裂变源 | 自发光 $L_e$ |

**图形学的简化**：光子无相互作用（线性）、仅表面碰撞、只求稳态。

**图形学的困难**：场景几何极复杂（数十亿图元）、需求"点探测器"（像素）精确值。

### 11.2 分裂 (Splitting)

**效率度量**：$\epsilon = 1/(\sigma^2 \tau)$（效率 = 1 / (方差 × 单样本成本)）

- 路径追踪：每次碰撞只选一条后继（低成本，高方差）
- 光线树：每次分裂多条子射线（高成本，低方差）

**最优策略**：自适应混合 — 在直接光源/高贡献区域分裂，在低贡献/长镜面链使用路径追踪+RR。

### 11.3 隐式俘获 (Implicit Capture)

表面反照率决定光子吸收概率。在蒙特卡洛中：**不直接终止粒子，而是降低权重并继续** → 统计更好。代价是需配合终止策略（如 RR）。

## 12. GPU 光线追踪

### 12.1 Brook for GPUs (2004)

**核心贡献**：将 GPU 抽象为**流处理器 (Stream Processor)**，是 CUDA/OpenCL 的先驱。

**流编程模型**：
- **Stream**：同类型数据集合（多维数组）
- **Kernel**：应用于流每个元素的并行函数
- **Reduction**：归约操作（需满足结合律）

**关键发现**：GPU 优于 CPU 的条件：计算强度 $\gamma > 1/(s-1)$，其中 $s = K_{cpu}/K_{gpu}$。

**局限性**：纹理精度限制、无 Scatter/Gather、归约需多 Pass → 后续被 CUDA 解决。

### 12.2 OptiX (2010)

> NVIDIA 首个生产级可编程 GPU 光线追踪引擎。

**7 种可编程程序**：
| 程序 | 功能 |
| :--- | :--- |
| Ray Generation | 入口，生成光线 |
| Intersection | 计算光线-几何体交点 |
| Bounding Box | 提供包围盒 |
| Closest Hit | 最近交点着色 |
| Any Hit | 每个交点调用（阴影/透明） |
| Miss | 未命中时调用 |
| Exception | 异常处理 |

**关键技术**：
- **Megakernel**：单一大内核包含所有程序
- **Continuation**：将递归转为状态机，解决 GPU 不支持递归调用的问题
- **细粒度调度**：同 SIMT 单元执行相同状态，减少控制流发散
- JIT 编译：PTX→PTX 变换，内联优化

**性能代价**：比手写 GPU 光线追踪器慢约 25–35%，但灵活性和易用性大幅提升。

### 12.3 现代硬件光线追踪

- **NVIDIA RTX**：专用 RT Core 硬件加速 BVH 遍历和三角形求交
- **Vulkan RT / DXR**：标准光线追踪 API
- **混合渲染**：光栅化 + RT（反射/阴影/GI）

## 13. 实时 GI 方案全景

| 方案 | 计算时机 | 视角无关 | 镜面 | 动态 | 质量 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Path Tracing** | 实时/离线 | 否 | ✅ | ✅ | 最高（有噪点） |
| **Radiosity** | 离线烘焙 | ✅ | ❌ | ❌ | 高（漫反射） |
| **Lightmap** | 离线烘焙 | 纹理烘焙 | ❌ | ❌ | 高（静态） |
| **Light Probes** | 离线+运行时插值 | 近似 | ❌ | ✅（物体） | 中（低频） |
| **SSGI/SSAO** | 实时（屏幕空间） | 否 | ❌ | ✅ | 中（有漏光） |
| **PRT** | 预计算+运行时 | ✅ | ❌ | 刚性旋转 | 中 |
| **Lumen (UE5)** | 实时（混合） | ✅ | ✅ | ✅ | 高 |

### 现代实时 GI 的趋势

- **UE5 Lumen**：屏幕追踪 + SDF 追踪 + 最终聚集的混合体，全动态、无需烘焙
- **RTXGI**：硬件 RT 加速的实时 GI 探针系统
- **混合渲染**：光栅化主体 + RT 补充反射/阴影/GI

## ⚠️ 常见考点

1. **渲染方程各部分含义？** → $L_e$：自发光；$f_r$：BRDF；$L_i$：入射光；$n \cdot \omega_i$：余弦衰减
2. **Whitted 模型与 Phong 模型的本质区别？** → Whitted 增加了反射和折射的递归项（全局光照）
3. **BVH vs KD-Tree 的优缺点？** → BVH 内存小、动态友好；KD-Tree 查询效率更高但构建慢
4. **路径追踪 vs Whitted 光线追踪的主要区别？** → 路径追踪每次只追踪一条后继路径（漫反射场景中更优）
5. **俄罗斯轮盘赌的无偏性？** → $\mathbb{E}(Y)=P\cdot Lo/P + (1-P)\cdot 0 = Lo$
6. **为什么需要光源采样（Next Event Estimation）？** → 半球均匀采样击中光源概率太低，效率极差
7. **重要性采样的最优 PDF？** → $p(x) \propto |f(x)|$
8. **蒙特卡洛积分的收敛率？** → MSE $\propto 1/N$，减半噪声需 4 倍样本
9. **OptiX 如何解决 GPU 不支持递归的问题？** → Continuation 状态机，将递归转为虚拟程序计数器驱动的循环
10. **离线 vs 实时 GI 的区别？** → 离线追求物理精确（Path Tracing），实时追求速度和近似的平衡（Lightmap/Probes/SSGI/Lumen）
