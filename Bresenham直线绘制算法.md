# Bresenham直线绘制算法 - 完整推导与实现

## 一、算法概述

Bresenham直线绘制算法是计算机图形学中最基础的栅格化算法之一，由Jack Bresenham在1962年提出。该算法的核心价值在于：**仅使用整数加法、减法和位移操作**，就能在像素网格上精确逼近任意直线，且无需浮点运算。

算法解决的问题很简单：给定两个端点坐标（整数），如何确定中间哪些像素应该被点亮，才能形成一条视觉上笔直的线。

**关键特性：**

- 增量式算法：每一步只做少量整数运算
- 无浮点运算：适合硬件实现和底层优化
- 误差可控：每个选择的像素都是当前列中离理想直线最近的像素

## 二、问题建模

### 2.1 约束条件

推导从最简情况开始：**第一象限内斜率 0 < m < 1，且 x1 < x2**。其他所有情况（不同斜率方向、八分圆）均可通过坐标变换对称处理。

这意味着：

- x方向每次步进1（主步进方向）
- y方向要么不变，要么增加1
- 问题简化为：**每走一列，y要不要+1？**

### 2.2 几何直觉

假设当前像素在 $(x_k, y_k)$，理想直线在列 $x_{k+1}$ 处的纵坐标为 $y$，则两个候选像素到理想位置的距离分别为：

- 选 $(x_k+1, y_k)$：$d_1 = y - y_k$
- 选 $(x_k+1, y_k+1)$：$d_2 = (y_k+1) - y$

**决策规则**：若 $d_1 \leq d_2$，选下方像素；否则选上方像素。

---

## 三、数学推导

### 3.1 从几何到决策参数

直线方程：$y = m \cdot (x_k+1) + b$，其中 $m = \Delta y / \Delta x$。

两距离之差：

$$d_1 - d_2 = 2m(x_k+1) - 2y_k + 2b - 1$$

为消去浮点数，定义决策参数 $p_k$，乘以 $\Delta x$（恒正，不改变符号）：

$$p_k = \Delta x \cdot (d_1 - d_2)$$

代入 $m = \Delta y / \Delta x$：

$$p_k = 2\Delta y \cdot x_k - 2\Delta x \cdot y_k + 2\Delta y + \Delta x(2b - 1)$$

$p_k$ 的符号直接决定选哪个像素：

- $p_k < 0$：$d_1 < d_2$，选下方像素
- $p_k \geq 0$：选上方像素

### 3.2 增量递推（核心优化）

我们不在每步重新计算 $p_k$，而是递推更新。对比 $p_{k+1}$ 和 $p_k$：

$$p_{k+1} = p_k + 2\Delta y - 2\Delta x \cdot (y_{k+1} - y_k)$$

由于 $(y_{k+1} - y_k)$ 只能是0或1：

- **选择下方像素** $(y$不变$)$：$p_{k+1} = p_k + 2\Delta y$
- **选择上方像素** $(y+1)$：$p_{k+1} = p_k + 2\Delta y - 2\Delta x$

这样每次迭代只需一次整数加法。

### 3.3 初始值

起点 $(x_0, y_0)$ 在直线上，代入推导可得：

$$p_0 = 2\Delta y - \Delta x$$

## 四、实现细节

### 4.1 第一象限（0 < m < 1）伪代码

```
plotLine(x0, y0, x1, y1):
    dx = x1 - x0
    dy = y1 - y0
    p = 2*dy - dx          // 初始决策值
    y = y0

    for x = x0 to x1:
        plot(x, y)
        if p < 0:
            p = p + 2*dy   // 选下方像素，y不变
        else:
            y = y + 1
            p = p + 2*dy - 2*dx   // 选上方像素
```

### 4.2 推广到任意斜率

需要处理八分圆的所有情况：

```java
public void drawLine(int x1, int y1, int x2, int y2) {
    int dx = Math.abs(x2 - x1);
    int dy = Math.abs(y2 - y1);
    int sx = (x1 < x2) ? 1 : -1;  // x方向步进
    int sy = (y1 < y2) ? 1 : -1;  // y方向步进
    int err = dx - dy;            // 初始误差
    
    while (true) {
        plot(x1, y1);
        if (x1 == x2 && y1 == y2) break;
        int e2 = 2 * err;
        if (e2 > -dy) {
            err -= dy;
            x1 += sx;
        }
        if (e2 < dx) {
            err += dx;
            y1 += sy;
        }
    }
}
```

这是最通用的实现形式，用一个误差变量 $err$ 统一处理所有斜率方向。

### 4.3 实现要点

1. **dx与dy取绝对值**：确定步进方向前，先取绝对值计算距离
2. **主步进方向判断**：若 |dx| > |dy|，以x为主步进；否则以y为主步进
3. **误差更新**：每次步进，误差调整量取决于选中的候选点

## 五、优化演进路径

从最朴素方法到最终算法，优化路径很清晰：

| 步骤 | 方法                                 | 问题                      |
| ---- | ------------------------------------ | ------------------------- |
| 1    | 每次用 `y = m*x + b` 计算            | 浮点乘除，慢              |
| 2    | 每次 `y += m` 增量更新               | 仍需浮点累加              |
| 3    | 将y拆为整数+小数部分，用`f += m`累加 | 小数比较 `f > 0.5`        |
| 4    | f乘以dx，用dy更新                    | 比较变为 `f > dx/2`       |
| 5    | f再乘以2                             | 比较变为 `f > dx`，无除法 |
| 6    | 初始 f = -dx（平移）                 | 比较变为 `f > 0`，更快    |

最终版本只需整数比较和加法，无任何乘除法在循环内。

## 六、数学证明（可选）

算法正确性的核心是：**每步选择的像素都是该列上离理想直线最近的整数点**。

对第一象限情形，可形式化证明：决策参数 $p_k$ 的符号与 $d_1-d_2$ 的符号完全一致（因 $\Delta x > 0$），而 $d_1-d_2 < 0$ 等价于下方候选点更近。

更严格的证明可用归纳法：$p_0$ 正确 ⇒ 每一步递推保持决策正确 ⇒ 所有点都在最优逼近路径上。

## 七、应用与扩展

- **圆与曲线绘制**：同样的误差累积思想可扩展到Bresenham圆算法
- **硬件实现**：因只用整数加减和位运算，适合GPU/FPGA等硬件加速
- **实时渲染**：现代管线虽然已由硬件完成栅格化，但理解该算法对理解光栅化原理仍有重要意义
