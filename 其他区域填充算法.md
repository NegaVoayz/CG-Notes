# 其他区域填充算法

## 一、种子填充算法（Seed Fill）

### 基本思想

从种子点开始，**递归扩展**到相邻像素，直到遇到边界。

### 两种规则

| 规则      | 扩展方向               | 特点                         |
| --------- | ---------------------- | ---------------------------- |
| **4连通** | 上下左右4个方向        | 速度快，但可能漏填对角线区域 |
| **8连通** | 上下左右+4个对角线方向 | 填充更完整，但可能溢出边界   |

### 算法流程

```
种子填充(种子点):
  ① 判断当前点是否在区域内 且 未填充
  ② 是 → 填充该点
  ③ 按连通规则访问相邻点
  ④ 递归执行 ①
```

### 优缺点对比

|              | 4连通          | 8连通             |
| ------------ | -------------- | ----------------- |
| **填充范围** | 十字形扩展     | 全方向扩展        |
| **边界处理** | 安全，不易溢出 | 可能穿过4连通边界 |
| **适用场景** | 标准图形填充   | 消除锯齿/复杂区域 |
| **递归深度** | 较浅           | 较深，易栈溢出    |

## 二、扫描线种子填充算法

**改进思路**：结合扫描线算法和种子填充

### 核心优化

- 将**逐点递归**改为**逐段填充**
- 填充一条水平区间后，再处理上下相邻行
- 减少递归次数，**避免栈溢出**

```
流程：
① 填充当前扫描线的一段连续区间
② 检查上下相邻行中的未填充段
③ 将新段的右端点作为种子入栈
④ 弹出种子，重复 ①-③
```

## 三、边界填充算法（Boundary Fill）

**思想**：从种子点向外扩展，遇到**指定边界色**停止

### 两种模式

| 模式         | 填充条件                 | 用途                 |
| ------------ | ------------------------ | -------------------- |
| **泛洪填充** | 替换所有相邻的**目标色** | 图片编辑（魔棒工具） |
| **边界填充** | 遇到**边界色**停止       | 封闭区域填充         |

---

## 四、算法对比总结

| 算法           | 适用场景       | 数据结构  | 复杂度    |
| -------------- | -------------- | --------- | --------- |
| **扫描线填充** | 多边形精确填充 | 边表(AET) | O(n×h)    |
| **4连通种子**  | 小区域/交互式  | 栈/队列   | O(面积)   |
| **8连通种子**  | 复杂纹理区域   | 栈/队列   | O(面积)   |
| **扫描线种子** | 大区域填充     | 栈        | O(分段数) |
| **泛洪填充**   | 图像编辑       | 栈/队列   | O(面积)   |

## 五、选择建议

| 场景               | 推荐算法       | 原因               |
| ------------------ | -------------- | ------------------ |
| CAD绘制多边形      | **扫描线填充** | 精确、可控、无递归 |
| 图像编辑（魔术棒） | **泛洪填充**   | 按颜色匹配自然     |
| 小游戏/像素画      | **4连通种子**  | 实现简单           |
| 大纹理区域         | **扫描线种子** | 内存友好           |
| 消除锯齿           | **8连通种子**  | 更自然的边界       |

---

**一句话总结**：扫描线适合**规则多边形**（效率优先），种子填充适合**任意形状区域**（灵活优先），两者在算法上可以结合使用。
