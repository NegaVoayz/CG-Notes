## 1. 纹理映射概述

**纹理映射**：将图像（纹理）贴到三维物体表面，增加真实感，避免用几何精确建模所有细节。

纹理映射的两个核心步骤：
1. **映射 (Mapping)**：确定 3D 表面点对应纹理上的哪个 2D 坐标
2. **滤波 (Filtering)**：当纹理坐标不是整数时，如何计算最终颜色

---

## 2. 纹理坐标与参数化

### 2.1 UV 映射（最主流）

- 美术在建模软件中将模型表面"剪开"并"摊平 (Flatten / Unwrap)"到 2D 图片
- 每个顶点获得 $(u, v)$ 坐标，$u, v \in [0, 1]$
- 光栅化阶段自动插值三角形内部的 UV 坐标

### 2.2 参数化 (Parameterization)

决定纹理坐标如何映射到几何表面：
- 投影映射 (Projection Mapping)（平面/圆柱/球面投影）
- 展开表面 (Surface Unfolding)
- 纹理图谱 (Texture Atlas)

### 2.3 其他映射方式

| 方式 | 用法 |
| :--- | :--- |
| **环境映射** | 反射/折射，通过法线方向去立方体贴图 (Cube Map) 采样 |
| **投影映射** | 像投影仪一样投射纹理（阴影贴图 (Shadow Map) 本质上就是投影纹理） |

## 3. 纹理滤波

> 纹理是离散的像素，即**纹素 (Texel)**，UV 坐标是连续的浮点数。当坐标落在像素之间时，如何取值？

### 3.1 放大滤波 (Magnification) — 纹理太小，物体太大

一个屏幕像素覆盖纹理上的一个小区域：

| 方法 | 效果 |
| :--- | :--- |
| **Nearest（最近邻）** | 马赛克 |
| **Bilinear（双线性插值）** | 取周围 $$2 \times 2$$ 个 Texel 加权平均，平滑但模糊 |

### 3.2 缩小滤波 (Minification) — 纹理太大，物体太小

一个屏幕像素覆盖纹理上的**一大片区域**。简单单点采样 → 闪烁和锯齿。

#### Mipmap

> **考点：** 核心考点。

预生成分辨率逐级减半的纹理金字塔：
- Level 0：原始分辨率
- Level 1：每维减半
- Level n：$1 \times 1$

存储量：$1 + 1/4 + 1/16 + \cdots = 4/3$ 倍原图。

**层级选择公式**（基于纹理坐标的屏幕空间导数 (Screen-space Derivative, ddx/ddy)）：
$$LOD（细节层次）= \log_2\left(\max\left(\sqrt{(du/dx)^2 + (dv/dx)^2}, \sqrt{(du/dy)^2 + (dv/dy)^2}\right)\right)$$

即选择使得纹素尺寸 $$\approx$$ 屏幕像素尺寸的 Mipmap 层级。

#### 三线性插值 (Trilinear Filtering)

在两层相邻的 Mipmap 层级之间再做一次双线性插值，消除层级切换时的"断层"。

### 3.3 局限性：各向异性 (Anisotropy)

Mipmap 假设像素足迹 (Pixel Footprint) 在纹理空间中是**正方形**，即各向同性 (Isotropic)。当观察角度很斜时（如从侧面看地面），足迹是**拉长的矩形或平行四边形**，正方形近似会导致过度模糊 (Overblur)。

## 4. 高级滤波技术

### 4.1 各向异性滤波 (Anisotropic Filtering)

沿观察方向**非均匀采样**，处理矩形或斜向像素足迹。

- 使用 **Ripmap**（水平和垂直分别 Mipmap）或多次采样
- 几乎为现代游戏画质标配
- 指定为各向异性过滤级别（如 $$4\times$$、$$8\times$$、$$16\times$$）

### 4.2 Summed Area Table (SAT)

每个像素存储**左下角所有像素的和**（积分图），用于快速计算任意矩形区域的平均颜色。

**查询公式**（矩形 $[x_0,y_0] \times [x_1,y_1]$）：
$$Avg = \frac{SAT(x_1,y_1) - SAT(x_0,y_1) - SAT(x_1,y_0) + SAT(x_0,y_0)}{(x_1-x_0)(y_1-y_0)}$$

- 支持各向异性（矩形区域）
- 存储量大（2~4 倍原图）
- 非 OpenGL 标准 (non-OpenGL standard)，少见于实时渲染

### 4.3 EWA 滤波 (Elliptical Weighted Average)

使用**椭圆高斯核**对不规则像素足迹做加权平均，质量最高但计算成本也最高。

### 4.4 滤波技术对比

| 技术 | 各向同性 | 各向异性 | 存储成本 | 质量 | 常见度 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| Nearest | 是 | 否 | $$1\times$$ | 最低 | 像素风格 |
| Bilinear | 是 | 否 | $$1\times$$ | 低 | 基础 |
| Mipmap + Trilinear | 是 | 否 | $$1.33\times$$ | 中 | 标准 |
| Anisotropic | 是 | 是 | $$1.33\times$$ | 高 | 游戏标配 |
| SAT | 是 | 是(矩形) | $$2{-}4\times$$ | 高 | 罕见 |
| EWA | 是 | 是(椭圆) | $$1.33\times$$ | 最高 | 离线 |

## 5. 纹理映射应用

| 应用 | 说明 |
| :--- | :--- |
| **调制纹理** (Modulation) | 用纹理值调节颜色或材质参数（木纹、砖墙） |
| **光照映射** (Light Map) | 纹理存储预计算的漫反射光照 |
| **环境映射** (Environment Map) | 立方体贴图模拟反射，视线方向+法线决定采样方向 |
| **凹凸映射** (Bump Mapping) | 扰动法线产生凹凸效果，不改变几何 |
| **位移映射** (Displacement Mapping) | 实际移动顶点 → 改变几何形状，需曲面细分 |
| **体积纹理** (Volume Texture) | 用 $(x,y,z)$ 索引纹理，可程序化生成（Perlin 噪声） |
| **基于图像的渲染** (IBR) | 真实照片映射到粗糙几何模型 |

## 6. 透视校正插值

### 问题
屏幕空间线性插值 UV → 3D 空间非线性。在透视投影下，屏幕上的匀速扫描不等于 3D 表面的匀速扫描。

### 数学原理

纹理坐标 $u$ 的透视校正插值：
$$u = \frac{(u/z)_{interp}}{(1/z)_{interp}}$$

即在屏幕空间对 $u/z$ 和 $1/z$ 做线性插值，再除回去。

**现代 GPU 自动完成透视校正**（`noperspective`（无透视校正）关键字可关闭）。

## 7. 凹凸/法线/位移贴图

> **考点：** 三者区别常考。

| 贴图类型 | 修改内容 | 几何改变 | 自遮挡 | 性能 |
| :--- | :--- | :--- | :--- | :--- |
| **凹凸贴图** (Bump Map) | 灰度图扰动法线 | 否 | 否 | 低 |
| **法线贴图** (Normal Map) | RGB 编码法线方向 | 否 | 否 | 低 |
| **位移贴图** (Displacement) | 移动顶点位置 | 是 | 是 | 高（需细分） |

### 法线贴图的切线空间

法线贴图通常存储**切线空间**中的法线（浅蓝色为主）：
- **切线空间**：以表面点为原点，切线-副切线-法线 (Tangent-Bitangent-Normal) 为基
- 优势：贴图可复用（同一砖墙法线贴图可用于不同朝向的墙面）
- 渲染时需将光照方向变换到切线空间（或反之）

## 常见考点

1. **Mipmap 解决了什么问题？如何选择层级？** → 解决了 Minification 时的走样（闪烁、摩尔纹）。基于纹理坐标的屏幕空间梯度选择层级。
2. **三线性插值的工作过程？** → 先在两层相邻 Mipmap 级别各做双线性插值，再在层级间线性插值。
3. **各向异性滤波为什么需要？普通 Mipmap 有什么局限？** → Mipmap 假设各向同性（正方形足迹），斜视角度的矩形足迹用正方形近似会导致过度模糊。
4. **SAT 的查询公式？** → $Avg = (SAT(x1,y1) - SAT(x0,y1) - SAT(x1,y0) + SAT(x0,y0)) / area$
5. **凹凸贴图 vs 法线贴图 vs 位移贴图的本质区别？** → 凹凸：灰度扰法线；法线：RGB 存法线；位移：实际改变顶点几何
6. **为什么法线贴图用切线空间？** → 贴图可跨不同朝向的几何面复用
7. **透视校正插值的原理？** → 对 $u/z$ 和 $1/z$ 线性插值，再除回去
