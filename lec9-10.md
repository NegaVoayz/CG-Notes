# 《计算机图形学》第9-10讲（光线追踪）

## 一、课程重点简述（Overall Summary）

本讲聚焦于**光线追踪（Ray Tracing）** 技术，涵盖：

- **Whitted 全局光照模型**（递归光线追踪，支持反射、折射、阴影）
- **光线与场景中几何体的求交算法**（球体、三角形、AABB等）
- **加速结构**（BVH、KD-Tree、Grid、Octree、BSP）
- **高级效果**：抗锯齿、软阴影、景深、分布式光线追踪
- **GPU光线追踪**（RTX、OptiX、Vulkan）
- **未来应用**：VR、移动端、Gaussian Splatting

## 二、详细笔记（核心概念深入分析）

---

### 1. Ray（光线）及其构建

#### 光线方程：
$$
\mathbf{R}(t) = \mathbf{E} + t(\mathbf{P} - \mathbf{E}), \quad t \in [1, +\infty)
$$
- **E**：视点（眼睛）
- **P**：像素中心在视平面上的位置
- **t=0**：在眼睛处，**t=1**：在像素处

#### 光线类型：
- **Eye Ray**：从眼睛穿过像素发射
- **Shadow Ray**：从交点射向光源，检测是否被遮挡
- **Reflection Ray**：根据反射定律生成
- **Refraction Ray**：根据Snell定律生成（需处理全反射）

#### 关键点：
- 每条光线递归地计算颜色
- 递归深度需限制（避免无限循环）

---

### 2. AA（Anti-Aliasing，抗锯齿）

#### 问题来源：
- 每个像素只采样一次（中心点）→ 高频信号导致**锯齿（aliasing）**
- 违反**奈奎斯特采样定理**（采样率需≥2倍最高频率）

#### 解决方法：**分布式光线追踪（Distributed Ray Tracing）**
- 每个像素发射**多条光线**（如 4、16、256 条）
- 采样点在像素区域内**随机抖动（jitter）**
- 结果取平均 → 锯齿转化为**噪声**（视觉上更可接受）

#### 扩展：
- 同样的方法可用于反射、折射方向的多重采样
- 结合**BRDF**或Phong模型加权

---

### 3. 半影区（Soft Shadows，软阴影）

#### 成因：
- 光源不是**点光源**，而是**面光源**
- 物体遮挡部分光源 → 产生**半影（penumbra）**

#### 实现方式：
- 从交点向**光源表面多个点**发射阴影光线
- 统计可见光源比例 → 决定光照强度
- 仍使用**Phong模型**计算漫反射和镜面反射，但系数乘以可见比例

#### 效果：
- 阴影边界模糊，更真实
- 需大量采样 → 计算成本高

---

### 4. 景深（Depth of Field，DOF）

#### 成因：
- 真实相机有**光圈**，不是针孔
- 焦点外的物体会模糊

#### 实现方式：
- 在**镜头（透镜）** 上随机采样多个点
- 每条光线从不同镜头位置出发，穿过同一像素点
- 聚焦平面上的物体所有光线汇聚于一点 → 清晰
- 非聚焦平面物体光线发散 → 模糊

#### 效果：
- 模拟摄影中的景深效果
- 增强视觉真实感

## 三、补充概念（快速回顾）

| 概念             | 说明                                      |
| ---------------- | ----------------------------------------- |
| **Whitted 模型** | Phong + 反射光 + 折射光                   |
| **反射与折射**   | 使用Snell定律、Fresnel近似（Schlick公式） |
| **加速结构**     | BVH、KD-Tree、Grid、Octree、BSP           |
| **GPU光线追踪**  | NVIDIA RTX、OptiX、Vulkan RT管线          |
| **应用场景**     | VR、游戏、电影、建筑设计                  |
